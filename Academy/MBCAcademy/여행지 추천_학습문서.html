01

프로젝트 개요

"어떤 사람에게 어떤 여행지를 추천할 수 있을까?"라는 질문에서 출발했다. 여행지를 자연·문화·체험 같은 카테고리로 나누고, 카테고리별 데이터를 분석해 조건(지역·유형)에 맞는 여행지를 골라주는 것이 목표. 데이터를 "받아쓰는" 것이 아니라 수집 → 정제 → 분석 → 결론의 전 과정을 직접 겪는 것에 의미를 두었다.

항목내용
데이터 소스공공데이터포털의 관광지 데이터 + 여행 관련 사이트 크롤링 데이터
분석 단위여행지 × 카테고리(자연·문화·체험 등) × 지역
결과물카테고리 조건을 넣으면 상위 여행지를 추천해주는 분석 노트북
담당 역할크롤링 코드 작성, 데이터 전처리, 카테고리 집계 분석

분석 파이프라인

P2전처리

결측치·중복 제거, 카테고리 표기 통일, 두 소스의 데이터 병합

P3분석

groupby 집계로 카테고리·지역별 특징 파악, 조건 필터링

P4도출

카테고리별 추천 리스트 정리, 시각화로 근거 제시

03

핵심 학습 개념

Pandas DataFrame

  • read_csv로 불러온 데이터를 표 형태로 조작
  • 불리언 인덱싱으로 조건에 맞는 행 추출
  • merge로 서로 다른 소스의 데이터 결합

데이터 수집 · 크롤링

  • 공공데이터포털에서 필요한 데이터 찾는 법부터 학습
  • requests + BeautifulSoup로 HTML에서 원하는 값 추출
  • 수집 데이터에는 반드시 정제 과정이 따라온다는 것을 체감

전처리

  • isna·dropna·fillna로 결측치 처리
  • 같은 의미의 다른 표기("자연", "자연경관")를 하나로 통일
  • 중복 행 제거와 자료형 변환

집계 · 분석 도출

  • groupby·pivot_table로 카테고리·지역별 집계
  • 정렬·상위 N 추출로 추천 리스트 생성
  • 수치를 그래프로 바꿔 근거 있는 결론 만들기
04

분석 노트

프로젝트의 흐름을 요약한 예시 코드. 실제 분석 노트북의 핵심 단계만 남긴 형태다.

In [1]크롤링 — 여행 사이트에서 여행지 이름·카테고리 수집
import requests
from bs4 import BeautifulSoup

rows = []
for page in range(1, 11):
    html = requests.get(url, params={"page": page}).text
    soup = BeautifulSoup(html, "html.parser")
    for item in soup.select(".spot-card"):
        rows.append({
            "name": item.select_one(".title").text.strip(),
            "category": item.select_one(".tag").text.strip(),
            "region": item.select_one(".region").text.strip(),
        })
In [2]전처리 — 결측치 제거와 카테고리 표기 통일
import pandas as pd

df = pd.concat([pd.DataFrame(rows), pd.read_csv("공공데이터_관광지.csv")])
df = df.dropna(subset=["name", "category"]).drop_duplicates("name")

category_map = {"자연경관": "자연", "문화재": "문화", "액티비티": "체험"}
df["category"] = df["category"].replace(category_map)
In [3]집계 — 카테고리·지역별 여행지 분석 후 추천 도출
summary = (df.groupby(["region", "category"])
             .agg(count=("name", "count"), score=("rating", "mean"))
             .reset_index())

# "자연" 카테고리에서 평점이 높은 지역 Top 5
top = (summary[summary["category"] == "자연"]
         .sort_values("score", ascending=False)
         .head(5))
05

배운 점 & 회고

  • 분석의 8할은 전처리. 실제 데이터는 표기가 제각각이고 비어 있는 값이 많아서, 분석보다 정제에 훨씬 많은 시간이 들었다.
  • 데이터는 직접 구할 수 있다. 공공데이터포털과 크롤링을 배우면서 "데이터가 없어서 못 한다"는 말이 줄었다.
  • groupby 하나로 질문이 풀린다. "어느 지역에 자연 여행지가 많은가?" 같은 질문이 집계 한 번으로 답이 되는 경험.
  • 결론에는 근거가 필요하다. 추천 리스트를 낼 때 수치와 그래프로 이유를 붙이는 습관을 들였다.
💡

한 줄 정리이 프로젝트에서 수집 → 전처리 → 집계 → 결론이라는 데이터 분석의 기본 루틴을 몸에 익혔고, 이 루틴은 이후 머신러닝 과정의 EDA·전처리로 그대로 이어졌다.
