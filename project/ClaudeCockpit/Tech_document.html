1프로그램 개요 · 현황

ClaudeCockpit은 여러 Claude Code 세션을 세션에 들어가지 않고 브라우저 대시보드에서 관찰·제어하는 로컬 관제 도구다. 새 터미널 멀티플렉서를 만들지 않고 wmux(Windows)/cmux(macOS) 위에 얇게 얹는 오케스트레이션 레이어로 동작한다.

항목내용
3계층 매핑관리자(대시보드) ─ 프로젝트(멀티플렉서 workspace) ─ 역할(workspace 안의 agent/pane)
진실의 원천root/<프로젝트>/project.json 폴더 선언(desired) ⊕ 멀티플렉서·프로세스 실측(actual)을 매 폴링마다 병합
정본 시스템cockpit/ (구 teamctl/은 리라이트로 폐기 — 재배선 금지)
전제 환경Windows 11(wmux) · macOS(cmux) · Node 20+(내장 모듈만) · git CLI · Claude Code(claude 명령)
보안 경계서버 127.0.0.1 바인드 + X-Cockpit-Token(GET / 제외 전부) · 원격 노출 비목표

현황 (v0.3.0 · 2026-07-19)

  • macOS(cmux) 지원 — mux 파사드 아래 cmux 드라이버 추가. 대시보드는 플랫폼을 모른 채 서버가 내려주는 mux 계약(ownsApp·canOpenWeb)만 따른다.
  • 활동 훅 자동 설치/수리(FS-7) — 서버 기동 시 ~/.claude/settings.json의 훅 커맨드를 절대경로로 실측, 미설치/옛 경로면 현재 리포 경로로 자동 재등록(멱등·비차단·백업).
  • workspace git 추적(FS-21) — 코크핏의 유일한 백그라운드 루프(2s). 활성 workspace가 바뀌면 그 저장소 페이지를 내장 브라우저 패널로 1회만 이동.
  • 사용량 표시(FS-16)는 제거됨(2026-07-15) — 로컬 트랜스크립트 합산이 /usage의 실제 한도(5시간·7일 창)와 구조적으로 불일치. 재도입하지 않는다.
  • 토큰 소비 0 원칙 — 관찰·제어는 전부 로컬 실측(파이프·프로세스·파일). 토큰이 도는 경로는 [▶ Claude 실행] 하나뿐이며 그것도 기동만 한다.

2시스템 아키텍처 — UML 컴포넌트 다이어그램

상위(server·lifecycle·bin)는 mux.js 파사드만 import 한다 — 플랫폼 분기는 파사드에서 끝난다. 모든 프로브(proc·ports·caps·git·activity)는 논블로킹 캐시라 /api/state 응답을 절대 막지 않는다.

component diagram — 실행 컴포넌트와 주요 호출 경로

핵심 데이터 계약은 GET /api/state = { projects, unlinked, ports, hookInstalled, mux } 하나다. 대시보드는 상태를 소유하지 않고 이 페이로드를 2.5초마다 폴링해 전체를 다시 그린다(상태 기반 렌더).

6상태 전이 — UML 상태머신

왼쪽: 프로젝트 status(선언). 오른쪽: 세션 3단계(실측). kill 경로는 개별(/kill-session)과 전체(/deactivate·confirm) 둘뿐 — 무명령 자동 종료 없음.

state machine — 프로젝트 생명주기 · 세션 3단계

채택(adopt)과 역할 해석 — 세션→역할 매칭 우선순위: adopted[agentId](채택) > 네임스페이스 label cockpit:<프로젝트>:<역할>(같은 프로젝트만) > plain label(구 형식 호환). 매칭 근거는 via(adopted / label-ns / label)로 페이로드에 실려 드로어에 표시된다.

9.2 신뢰성 기전

  • 캐시 계약 — 폴링·읽기는 getState()(stale 허용), 변이 결정은 getFresh()(실왕복), 변이 후 invalidate()/refreshState()(read-your-writes). stale로 스폰을 결정하면 중복 pane이 생긴다.
  • reconcile(자기치유) — 긍정 신호(제목/label 재발견)는 wsId 즉시 되쓰기, 부정 신호(실측 부재)는 연속 3스냅샷(GRACE) 관찰 후에만 해제. 오프라인 스냅샷에선 판정하지 않고, 재연결(epoch 변화) 시 카운트를 리셋한다.
  • label 네임스페이스 — 스폰 label은 cockpit:<폴더>:<역할>. wmux 재시작으로 wsId·agentId가 전부 갈려도 label만 살아있으면 프로젝트·역할이 복원된다. env(COCKPIT_PROJECT/ROLE) 주입은 훅 오귀속 방지의 2차 신원.
  • 낙관적 재검증 — kill류(세션·포트)는 실행 직전 fresh 실측과 대조해 불일치면 409(화면이 낡음). 엉뚱한 대상 kill을 구조적으로 차단.
  • 우아한 성능 저하 — 멀티플렉서 다운 시 offline/demo 배지, 프로브 실패 시 해당 표시만 생략. 어떤 프로브도 폴링 응답을 막지 않는다.

11보안 · 불변 규칙

보안

  • 서버는 127.0.0.1 전용 + 토큰(config.token, gitignore) — 방화벽 허용 불필요, 원격 노출 비목표.
  • POST /open은 root/ 밖 경로 400(경로 탈출 방지) · 링크/open-web은 http(s)만.
  • 시크릿(.env·deploy-keys·connections.json)은 격리 .gitignore가 선제 차단 — 값은 저장·표시하지 않는다(키 존재만).
  • git 명령은 비대화 강제(GIT_TERMINAL_PROMPT=0·SSH BatchMode) — 자격증명 프롬프트로 서버가 멈추지 않는다.

불변 규칙 (어기면 재발하는 버그 — 코드 실측 확정)

  1. 읽기=getState() · 변이 결정=getFresh() · 변이 후 invalidate() — stale 스폰 결정 금지.
  2. agent kill은 리스트에서 안 지워진다 → 파사드의 isDead 중앙 필터가 거른다.
  3. 세션 전송은 surfaceId 명시 필수 — 포커스 기반 전송은 오발송.
  4. 경로 진실 = 폴더 — 스폰엔 항상 cwd 명시(드리프트 차단).
  5. git 저장소는 ops/ 에만 — 프로젝트 루트·상위 cockpit 저장소는 대상 아님.
  6. kill은 두 명시 경로뿐(개별·전체 confirm) — 무명령 자동 종료 없음.
  7. 세션 활동은 Claude 훅 실측 — claude on일 때만 노출 · working 10분 stale 방어.
  8. 논블로킹 프로브 — 어떤 프로브도 /api/state를 막지 않는다.
  9. 격리 스캐폴드 — clone·import된 코드 폴더에 cockpit 파일 미주입, 기존 파일 미덮어쓰기.
  10. 대시보드는 토큰 프리 — 폴링·표시가 Claude API를 호출하는 구조를 만들지 않는다(사용량 표시 재도입 금지).

12.3 알려진 경계 · v1 비목표

  • cmux openWeb 미구현 — 여는 CLI/RPC 계약이 미문서. macOS에서 실측 후 드라이버에 추가하면 상위 코드 변경 없이 git 칩 내장 패널·FS-21 추적이 함께 켜진다.
  • 크로스 볼륨 연동 미지원 — /import 이동은 동일 볼륨 rename만(EXDEV는 안내 후 원본 무변경).
  • darwin 포트 kill의 손자 고아 가능성 — pkill -P+kill은 taskkill /T와 달리 손자를 못 잡는다(알려진 차이).
  • 원격 노출·인증 확장 · DB/배포 조작 · git diff 뷰 · 트랜스크립트 파싱(훅 모델 판별만 예외) · 사용량 표시(제거됨) — 전부 범위 밖.
