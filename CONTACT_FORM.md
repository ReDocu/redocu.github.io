# 구글 문의 폼 문항 구성

메인 페이지 Contact 섹션(`index.markdown` `#contact`)에서 연결되는 **한국어 구글 폼**의 문항 정의서.
폼을 수정하면 이 문서도 함께 갱신한다. 폼 주소는 `_config.yml`의 `contact_form_url`에 넣는다.

영문 폼은 [CONTACT_FORM.en.md](CONTACT_FORM.en.md)에 따로 정의되어 있고 주소는 `contact_form_url_en`에
넣는다. 두 폼은 문항 구성이 동일하므로 **한쪽을 고치면 반드시 다른 쪽도 함께 고친다.** 설계 원칙과
주의사항은 이 문서에만 두고, 영문 문서는 입력할 내용만 담는다.

## 설계 원칙

공개 포트폴리오 사이트의 문의 폼은 **작성 도중 이탈**이 가장 큰 실패 요인이다. 특히 헤드헌터와
채용 담당자는 하루에 수십 명에게 연락하므로 문항이 길면 그냥 닫는다. 그래서 다음을 지킨다.

1. **첫 화면은 3문항.** 문의 유형만 고르면 나머지는 유형별 섹션으로 분기시킨다.
2. **유형별 필수는 4개 이하.** 나머지는 선택으로 두고, 없으면 회신 때 되물으면 된다.
3. **연봉·예산은 선택으로.** 필수로 걸면 이탈률이 크게 오른다. 대신 "범위로 적어도 된다"고 안내해
   심리적 부담을 낮춘다.
4. **파일 업로드 문항은 쓰지 않는다.** 아래 주의사항 참고.

## 폼 설정 (문항보다 먼저)

| 설정 위치 | 값 | 이유 |
| --- | --- | --- |
| 설정 > 응답 > 이메일 주소 수집 | `응답자 입력` | 회신 주소를 자동 확보. 별도 이메일 문항이 불필요해진다 |
| 설정 > 응답 > 응답 사본 전송 | `요청 시` | 상대가 보낸 내용을 스스로 보관할 수 있어 신뢰도가 오른다 |
| 설정 > 응답 > 로그인 필요 | **해제** | 켜면 구글 계정 없는 담당자가 아예 못 쓴다 |
| 설정 > 응답 > 응답 편집 허용 | 켬 | 오타·누락을 상대가 직접 고칠 수 있다 |
| 설정 > 프레젠테이션 > 진행률 표시줄 | 켬 | 분기 폼이라 남은 분량이 보여야 이탈이 준다 |
| 응답 탭 > 스프레드시트 연결 + 이메일 알림 | 켬 | 문의를 놓치지 않는다 |

### 주의: 파일 업로드 문항

구글 폼에 **파일 업로드 문항을 하나라도 넣으면 폼 전체가 구글 로그인 필수로 바뀐다.**
채용 담당자·헤드헌터에게 이건 치명적인 마찰이다. 스크린샷이나 채용 공고 파일이 필요하면
업로드 대신 **링크를 붙여 넣는 단답 문항**으로 받는다.

### 주의: 스팸

구글 폼에는 캡차가 없고, 이 폼 주소는 공개 사이트에 노출된다. 스팸이 들어오기 시작하면
순서대로 대응한다: ① 장문 필수 문항 유지(봇이 못 채운다) → ② 응답 알림으로 수동 필터 →
③ 그래도 심하면 `로그인 필요`를 켠다. 처음부터 ③으로 가지 않는다.

## 제작 순서 요약

1. 구글 폼 새로 만들기 → 제목 `DOCU 문의하기`
2. 위 **폼 설정** 표대로 설정 먼저 잡기
3. 섹션 1~8을 순서대로 만들기 (섹션을 다 만든 뒤에 분기를 건다)
4. 섹션 1의 문의 유형 문항에서 `답변을 기준으로 섹션 이동` 설정
5. 섹션 2~7 각각의 하단에서 `섹션 8로 이동`으로 지정
6. 미리보기로 6개 경로를 전부 한 번씩 통과해 보기
7. `보내기 > 링크`에서 URL 복사 → `_config.yml`의 `contact_form_url`에 붙여넣기 → Jekyll 재시작
