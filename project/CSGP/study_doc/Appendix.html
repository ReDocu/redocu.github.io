부록 — 용어집 · 핵심 이슈 · 요약

이 부록은 책 전체에서 나온 낯선 단어, 되풀이되는 함정과 주의사항, 그리고 한눈에 보는 요약을 한곳에 모은 참조 문서다. 본문을 읽다 모르는 말이나 헷갈리는 규칙을 만나면 여기서 찾아보면 된다.

  • 부록 A. 용어집 — 개념을 한 줄로 정의하고, 처음 나오는 장을 표시한다.
  • 부록 B. 핵심 이슈·함정 — 초보자가 꼭 한 번 걸려 넘어지는 지점과 그 해결.
  • 부록 C. 한눈에 요약 — 로드맵·복선 사슬·핵심 패턴 치트시트.

장 표기: 1-A=1부 A모듈, 1-9장=1부 9장, 2-0=2부 0단계 엔진, 2-5=2부 5단계 Galaga.

부록 A. 용어집 (Glossary)

A.1 값 · 연산 · 제어 흐름 (1부 A·B 모듈)

용어 뜻 처음
변수(variable) 값을 담는 "이름표 붙은 상자". 선언해서 만들고 대입으로 값을 넣는다 1-1장
자료형(data type) 상자의 모양. int(정수)·double(실수)·char(글자 하나) 1-1장
초기화 변수를 선언하면서 동시에 값을 넣는 것 (int m = 5000;) 1-1장
const "이 값은 바뀌지 않는다"고 못 박는 상수 선언 1-1장
형변환(캐스트) 정수↔︎실수처럼 자료형을 바꾸는 것. (double)a 1-1장
서식 지정자 printf 의 자리표시자. %d(정수) %s(문자열) %.1f(소수1자리) %5d(폭) 1-2장
이스케이프 문자 \ 로 시작하는 특수 표기. \n(줄바꿈) \t(탭) %%(퍼센트) 1-2장
정수 나눗셈 정수끼리 / 는 몫만 남고 소수점을 버린다 (7/2=3) 1-3장
나머지 연산자 % 나누고 남은 나머지 (17%5=2). 확률·순환·자릿수에 쓰임 1-3장
복합 대입 money += x = money = money + x (증감 ++ 포함) 1-3장
비교/논리 연산자 == != < >= (비교) / &&(그리고) ||(또는) !(아니다) 1-3장
연산자 우선순위 곱·나눗셈이 덧·뺄셈보다 먼저. 헷갈리면 괄호로 1-3장
조건문 if / else if / else — 조건이 참인 갈래 하나만 실행 1-4장
switch / case 값에 따라 여러 갈래로. 각 case 끝에 break 필수 1-4장
삼항 연산자 조건 ? A : B — 값 하나를 한 줄로 고르기 1-4장
반복문 while(조건 동안) for(정해진 횟수) do-while(최소 1번) 1-5장
break / continue 반복 즉시 종료 / 이번 회차만 건너뛰기 1-5장
무한 루프 while(1) / for(;;) — 항상 참, 안에서 break 로 탈출 1-5장
함수 이름 붙인 코드 묶음. 반환형·이름·매개변수·몸통 1-6장
매개변수 / 반환값 함수에 넘기는 입력 / 함수가 돌려주는 출력 1-6장
void 반환값이 없는 함수의 반환형 1-6장
스코프(scope) 변수가 사는 범위. 지역(함수 안)/전역(함수 밖, 모두 접근) 1-6장
값에 의한 전달 함수에 넘긴 값은 복사본이라 원본을 못 고침 → 포인터 필요 1-6장

A.2 배열 · 문자열 · 데이터 (1부 C·D 모듈)

용어 뜻 처음
배열(array) 같은 자료형 여럿을 연속 칸에 담고 인덱스로 접근 1-7장
인덱스 배열 칸 번호. 0부터 시작 (크기 4면 0~3) 1-7장
2차원 배열 배열의 배열 (표). board[H][W], 이중 for 로 순회 1-7장
병렬 배열 한 대상의 정보를 여러 배열에 나눠 담기(불편) → 구조체로 해결 1-7장
문자열 char 배열 + 끝의 널 문자 '\0'. 크기 = 글자수+1 1-8장
널 종료 '\0' "문자열 끝" 표시. printf 는 이걸 만나면 멈춤 1-8장
string.h 문자열 도구. strcpy(복사) strlen(길이) strcmp(비교) 1-8장
fgets 공백 포함 한 줄 입력 (scanf %s 는 공백에서 끊김) 1-8장
구조체(struct) 서로 다른 값 여럿을 하나로 묶는 사용자 정의 자료형 1-9장
멤버 구조체 안의 각 값. .(변수) 또는 ->(포인터)로 접근 1-9장
typedef 자료형에 짧은 별명 붙이기 (typedef struct{…} Employee;) 1-9장
enum(열거형) 이름 붙인 정수 목록 (G_RPG=0, G_ACTION=1 …) 1-9·15장
구조체 배열 같은 구조체 여럿 (Employee staff[4]) — 병렬 배열을 대체 1-9장

A.3 포인터 · 동적 메모리 (1부 D·E 모듈)

용어 뜻 처음
포인터(pointer) 주소를 담는 변수. int *p = int가 사는 주소를 담음 1-10장
주소 연산자 & 변수가 사는 위치. &hp = hp의 주소 1-10장
역참조 * 포인터가 가리키는 곳의 값. *p = 100 이면 원본이 바뀜 1-10장
-> (화살표) 구조체 포인터의 멤버 접근. p->x = (*p).x 1-10장
주소에 의한 전달 함수에 주소를 넘겨 원본을 직접 고침 (값 전달의 반대) 1-10장
NULL "아무것도 안 가리킴". 역참조하면 프로그램이 죽음 → 검사 필수 1-10장
스택 vs 힙 스택(지역변수·자동 소멸) / 힙(실행 중 크기 결정·직접 해제) 1-11장
malloc / free 힙에서 메모리 잡기 / 반납. sizeof 로 바이트 계산 1-11장
realloc 잡은 배열을 더 크게. 임시 포인터로 받아 실패 대비 1-11장
메모리 누수 free 를 안 해 붙잡힌 채 못 쓰는 메모리 낭비 1-11장
댕글링(dangling) 이미 해제/재사용된 것을 옛 포인터·인덱스로 가리킴 1-11장·2-5

A.4 랜덤 · 파일 · 모듈 (1부 E·F 모듈)

용어 뜻 처음
rand / srand / time 난수 뽑기 / 씨앗 심기 / 현재 시각. srand 는 한 번만 1-12장
범위 난수 min + rand() % 개수 (주사위 = 1 + rand()%6) 1-12장
확률 분기 rand() % 100 < 확률 로 "N% 확률" 표현 1-12장
시드(seed) 난수의 출발점. 같은 시드 = 같은 난수 순서(재현 가능) 1-12장
fopen / fclose 파일 열기/닫기. 모드 "r"(읽기) "w"(쓰기·덮어씀) 1-13장
fprintf / fscanf printf/scanf 의 파일 버전. 쓴 순서 그대로 읽어야 함 1-13장
텍스트 vs 바이너리 사람이 읽는 글자 / 메모리 바이트 그대로(작지만 안 읽힘) 1-13장
헤더(.h) / 소스(.c) 선언(명세) / 정의(구현). 큰 코드를 파일로 나눔 1-14장
인클루드 가드 #ifndef/#define/#endif — 같은 헤더 중복 포함 방지 1-14장
extern 다른 파일에 정의된 전역 변수를 공유 선언 1-14장
#define 매크로. 매직 넘버(맥락 없는 숫자)에 이름 붙이기 1-15장

A.5 C++ · 객체 · 엔진 (2부 0단계)

용어 뜻 처음
클래스 / 메서드 데이터+동작을 묶은 것 / 클래스 안의 함수 2-0
상속 부모 클래스의 것을 물려받아 확장 (: public IContent) 2-0
가상 함수 virtual 자식이 재정의할 수 있는 함수. 다형성의 핵심 2-0
순수 가상 = 0 몸통 없이 선언만. 자식이 반드시 구현 → 추상 클래스 2-0
가상 소멸자 IContent* 로 자식을 delete할 때 자식 소멸자가 불리게 2-0
인터페이스 구현 없이 "지켜야 할 메서드"만 정한 계약 (IContent) 2-0
템플릿 타입을 나중에 정하는 틀 (template<typename T>, Pool<T,N>) 2-0
정적 멤버 static 클래스마다 하나만 존재하는 멤버 (싱글턴 인스턴스) 2-0
싱글턴(singleton) 게임 전체에 딱 하나만 존재하는 객체 (매니저 4종) 2-0

용어 뜻 처음

더블 버퍼링 백버퍼에 다 그린 뒤 한순간에 교체 → 깜빡임 제거 2-0

엣지 / 레벨 입력 OnKeyDown(막 눌린 순간 1번) / OnKeyStay(눌린 동안 계속) 2-0

HUD 점수·목숨 등을 보여주는 정보 표시 영역 2-4

A.7 게임 기법 (2부 전반)

용어 뜻 처음

AABB 축 정렬 사각형 겹침 판정. !(오른쪽<왼쪽 || …) 2-3
히트박스 충돌 판정에 쓰는 상자. 그림보다 관대하게(작게) 잡기도 2-3
월드 스크롤 플레이어 고정, 세계가 흘러 달리는 착시 2-3
시차(parallax) 먼 것은 느리게 스크롤해 원근(깊이) 표현 2-3
물리 적분 매 프레임 v += 중력; y += v 로 부드러운 운동 2-3
절차적 생성 미리 배치 않고 규칙+난수로 그때그때 생성 2-3·2-6
게이팅(gating) 점수·시간·스테이지 조건으로 난이도 요소를 여는 것 2-3·2-4
엔티티 시스템 여러 종류를 하나의 struct+type으로 한 리스트에 관리 2-4
리소스 관리 연료·목숨처럼 소모/회복하는 자원, 0이면 실패 2-4
무적 타이머 상태에 수명을 줘 시간이 지나면 저절로 해제 2-4
스프라이트 여러 줄 문자로 그린 그림. 프레임 토글로 애니메이션 2-3
DAS Delayed Auto Shift — 꾹 누르면 초기 지연 후 자동 반복 2-2
라인 클리어 꽉 찬 행을 지우고 위를 끌어내림(배열 shift) 2-2
월킥(wall kick) 회전 시 벽에 걸리면 밀어 놓을 자리를 찾음 2-2
7-bag 7조각을 섞어 하나씩 꺼내 공정하게 배분 2-2
편대(formation) 슬롯 좌표만 저장하고 위치는 파생 (formX 하나로 전체 이동) 2-5
오브젝트 풀 고정 배열을 미리 잡고 active 플래그로 재사용, 힙 할당 0회 2-5
active 플래그 풀 슬롯이 쓰이는 중인지 표시. Kill=false, Spawn=true 2-5
측정 후 최적화 단순·정확 먼저, 실제 느린 곳을 잰 뒤에 개선 2-5
파괴 지형 총알에 부서지는 타일(벽돌) — 정적 맵이 동적으로 변함 2-7

A.8 알고리즘 · 자료구조 (2부 Maze·Battle City)

용어 뜻 처음
큐(queue) 먼저 넣은 것이 먼저 나옴(FIFO). BFS가 씀 2-6
스택(stack) 나중에 넣은 것이 먼저 나옴(LIFO). DFS·백트래킹이 씀 2-2·2-6
BFS 너비 우선 탐색. 가까운 것부터 균등 확장 → 무가중 최단 2-6
DFS 깊이 우선 탐색. 한 방향 파고들다 백트랙 (최단 아님) 2-6
A* f = g + h. 휴리스틱으로 목표 쪽 편향 → 적게 탐색·최단 2-6
휴리스틱 / 맨해튼 거리 남은 거리 추정 / |dx| + |dy| (격자 거리) 2-6
프론티어 미로 안과 밖의 경계 후보 집합 (Prim 생성) 2-6
백트래킹 막다르면 갈림길로 되돌아감 (스택 pop) 2-6
union-find(분리집합) 원소가 어느 집합인지 빠르게 판정·합치기 2-6
경로 압축 Find 시 부모 사슬을 짧게 만들어 다음 조회 가속 2-6
MST(최소 신장 트리) 모든 정점을 순환 없이 잇는 최소 간선 집합 (Kruskal) 2-6
분할정복 큰 문제를 작은 문제로 나눠 재귀적으로 풀기 2-6
flood fill 시작칸에서 갈 수 있는 곳으로 번져 연결 영역 채우기 2-7
최단경로 복원 prev 를 목표→시작으로 거꾸로 따라 경로 재구성 2-6
그리디(greedy) 매 순간 당장 최선인 선택 (완벽 최단은 아닐 수 있음) 2-7
편향(bias) 알고리즘의 규칙이 결과를 한쪽으로 쏠리게 함 (Binary Tree 미로) 2-6

C.5 학습 원칙 (책이 되풀이한 것)

  1. 불편을 먼저 겪고 도구를 쥔다. 병렬 배열의 고통 뒤 구조체, new/delete 남발 뒤 오브젝트 풀.
  2. 동작하는 단순함을 먼저, 최적화는 측정한 뒤에. 선형 탐색·그리디 AI도 "지금 충분한가"로 판단.
  3. 완성된 것에도 성장 여지가 있다. as-built 이슈는 결함이 아니라 다음 단계로 가는 문.
  4. 배우는 순서가 곧 만드는 순서다. 그래서 "이걸 어디 쓰지?"라고 묻지 않는다.
