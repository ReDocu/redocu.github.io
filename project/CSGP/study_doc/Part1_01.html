A 모듈 · 1–4장 — 값·계산·분기: 회사에 숫자를 세우다

모듈 A · 1–4장 · 게임 버전 v0.1 → v0.2 · 예상 학습 시간 3–5시간 처음 배우는 기술 — int double char const 형변환 printf 서식 이스케이프 문자 scanf 산술·비교·논리 연산자 정수 나눗셈·% 연산자 우선순위 if / else if switch 삼항 연산자

모든 프로그램은 결국 세 가지를 한다. 값을 담고, 계산하고, 갈라진다. 이 모듈에서 그 셋을 배우며, 우리 게임 회사의 첫 화면 — 자본금과 날짜가 찍힌 현황판 — 을 세우고, 한 달치 정산을 계산하고, 자본금이 바닥나면 파산을 선고하는 데까지 간다. 아직 게임은 "돌지" 않는다(반복문은 B 모듈이다). 하지만 이 모듈이 끝나면, 우리 손에는 이미 한 턴짜리 게임의 뼈와 살이 쥐어져 있다.

이 모듈의 학습 목표

이 모듈을 마치면 다음을 할 수 있다.

  1. int · double · char 의 차이를 알고, 게임 상태(자본금·명성·날짜)를 알맞은 자료형의 변수에 담을 수 있다.
  2. printf 의 서식 지정자(%d %s %.1f 폭 지정)로 표 형태의 화면을 정렬해 출력하고, scanf 로 정수를 입력받을 수 있다.
  3. 정수 나눗셈(/)과 나머지(%) 의 동작을 설명하고, 세금·거스름돈 같은 계산에 쓸 수 있다.
  4. 연산자 우선순위를 의식해 정산 수식(수입 − 세금 − 임대료)을 정확히 짤 수 있다.
  5. if / else if / else 와 switch 로 갈래를 나누고, 삼항 연산자로 간단한 선택을 한 줄에 쓸 수 있다.
  6. 자본금이 음수가 되는 순간을 if 로 잡아 파산(게임 오버) 을 판정할 수 있다.

왜 여기서부터 시작하는가

이 책은 게임을 키우는 데 필요한 순서로 문법을 배운다. 그렇다면 왜 값·계산·분기가 맨 처음일까?

게임을 아주 멀리서 보면, 이렇게 생겼다.

상태를 담는다  →  상태를 바꾼다  →  조건에 따라 갈라진다  →  (반복)
   (변수)          (연산자)          (조건문)              (반복문·B모듈)

우리 타이쿤도 다르지 않다. 자본금이라는 값이 있고, 매달 그 값이 정산으로 바뀌며, 자본금이 0 밑으로 떨어지면 파산으로 갈라진다. 마지막의 "반복"만 빼면 — 그 반복(월 루프)은 B 모듈의 몫이다 — 게임의 심장 박동 한 번이 이 모듈 안에 다 들어 있다.

그래서 순서가 이렇다. 담을 그릇(변수)이 없으면 아무것도 시작되지 않으니 1장은 변수다. 담았으면 보여 주고 받아야 하니 2장은 입출력이다. 값이 있으면 계산해야 하니 3장은 연산자, 계산 결과로 길이 갈라져야 하니 4장은 조건문이다. 네 장이 끝나면 "한 달"이 계산되고, B 모듈에서 그 한 달을 while 로 감싸는 순간 게임이 살아 움직이기 시작한다.

이 모듈에서 만드는 것. 실행하면 회사 현황판이 한 장 뜨고(v0.1), 시작 자본을 입력받아 한 달을 정산한 뒤, 메뉴에서 고른 번호에 따라 한 번 갈라지고, 자본금이 음수면 파산을 선고하는(v0.2) 한 턴짜리 프로그램이다. 아직 while 이 없어 "한 번만" 돌지만, 그 한 번 안에 게임의 모든 기본기가 들어 있다.

1장 · 변수와 자료형 — 회사를 숫자로 세우다

1.1 값을 담는 상자, 변수

프로그램이 다루는 모든 것은 결국 값이다. 자본금 5000, 올해 2026, 명성 0 … 이 값들을 어딘가에 담아 두고 이름으로 부르기 위해 쓰는 것이 변수(variable) 다. 변수는 "이름표가 붙은 상자"라고 생각하면 된다.

int money;        /* money 라는 이름의 정수 상자를 하나 만든다 (선언) */
money = 5000;     /* 그 상자에 5000 을 넣는다 (대입) */

선언과 대입을 한 줄에 합칠 수도 있다. 이걸 초기화라 한다.

int money = 5000;   /* 상자를 만들면서 곧바로 5000 을 넣는다 */

변수 이름은 그 값이 무엇인지 말해 주어야 한다. int m = 5000; 보다 int money = 5000; 이 낫고, int y = 2026; 보다 int year = 2026; 이 낫다. 코드는 사람이 읽는 글이기도 하다.

1.2 자료형 — 상자의 모양: int, double, char

상자에는 모양이 있다. 정수만 담는 상자, 소수를 담는 상자, 글자 하나를 담는 상자가 다르다. 그 모양을 자료형(data type) 이라 한다. 우리 게임에 필요한 셋만 먼저 익히자.

자료형 담는 것 예시 우리 게임에서
int 정수 (소수점 없음) 5000, -100, 0 자본금·명성·연도·월 — 거의 전부
double 실수 (소수점 있음) 7.5, 0.0 게임 평점(0.0 ~ 10.0)
char 글자 하나 'A', '#' 메뉴 문자, 게이지 칸

우리 게임의 돈 단위는 만원이고 전부 int 로 센다. "4,350만원"을 4350이라는 정수로 다루면, 큰 수의 자릿수 문제 없이 오직 정수 연산만으로 회사 경제가 돌아간다. 소수가 필요한 건 사실상 게임 평점 하나뿐이라, 그것만 double 을 쓴다.

int    money  = 5000;    /* 자본금: 5000만원 */
int    fame   = 0;       /* 명성: 0에서 시작 */
int    year   = 2026;    /* 연도 */
int    month  = 1;       /* 월 */
double rating = 0.0;     /* 게임 평점 — 유일하게 실수가 필요한 값 */

정수 하나가 8칸을 넘지 않게. int 는 보통 약 ±21억까지 담는다. 우리 돈 단위가 "만원"인 것은 이 한계와도 관련이 있다. 1억을 "만원 단위"로 세면 10000이라 넉넉하지만, "원 단위"로 세면 1억(=100000000)이 되어 자릿수가 커진다. 단위를 잘 고르면 자료형이 편해진다 — 이것도 설계다.

1.3 const — 바뀌면 안 되는 값

어떤 값은 게임 내내 절대 바뀌지 않는다. 사무실 임대료 100만원, 시작 자본 5000만원 같은 규칙 상수가 그렇다. 이런 값에는 const 를 붙여 "이건 상수다, 바꾸지 마라"고 못 박는다.

const int RENT       = 100;    /* 사무실 임대료 (만원/월) — 절대 안 변함 */
const int START_MONEY = 5000;  /* 시작 자본 */

RENT = 200;   /* ✗ 컴파일 에러! const 는 바꿀 수 없다 */

상수에 이름을 붙이면 두 가지가 좋아진다. 첫째, 코드에 흩어진 100 이 무슨 뜻인지 분명해진다(money - 100 보다 money - RENT). 둘째, 임대료를 바꾸고 싶을 때 const int RENT 한 줄만 고치면 된다. 이렇게 "값에 이름을 붙여 한곳에서 관리"하는 습관은 15장의 밸런싱 작업에서 빛을 발한다. (C에는 #define RENT 100 이라는 또 다른 상수 표기도 있는데, 그건 F 모듈에서 다룬다.)

1.4 형변환 — 정수와 실수를 섞을 때

정수와 실수를 함께 계산하면, 값이 자동으로 한쪽 모양으로 바뀐다. 이걸 형변환(type conversion) 이라 한다. 여기엔 초보자가 꼭 한 번 걸려 넘어지는 함정이 있다.

double half = 7 / 2;      /* 3.0  — ?! 3.5 가 아니다 */

왜 3.5가 아니라 3.0일까? 7 과 2 는 둘 다 정수라, 7 / 2 는 정수 나눗셈으로 먼저 계산되어 3이 된다(나머지 1은 버려짐). 그 3을 double 상자에 담으니 3.0이 될 뿐이다. 소수까지 얻으려면, 나누기 전에 한쪽을 실수로 만들어야 한다.

double half = 7.0 / 2;    /* 3.5 — 7.0 이 실수라 실수 나눗셈이 된다 */
double r = (double)quality / need;   /* (double) 로 강제 변환 후 나눗셈 */

(double)quality 처럼 괄호 안에 자료형을 적어 값을 강제로 바꾸는 것을 명시적 형변환(캐스트) 이라 한다. 게임 평점을 낼 때 "품질 ÷ 필요치"를 소수로 구하려면 이 캐스트가 필요하다 — 이 계산은 D 모듈의 출시 로직에서 실제로 쓰인다. 지금은 "정수끼리 나누면 소수점이 잘린다"는 사실 하나만 몸에 새겨 두자. 3장에서 이 성질을 오히려 이용해 세금을 계산할 것이다.

2장 · 표준 입출력 — 현황판을 그리고 자본을 입력받다

값을 담았으니, 이제 보여 주고(출력) 받는다(입력). 이 장이 끝나면 게임의 첫 화면인 회사 현황판이 완성된다(v0.1).

2.1 printf — 서식 있는 출력

printf 는 화면에 글자를 찍는다. 그냥 문자열을 넘기면 그대로 나오고, 값을 끼워 넣고 싶으면 서식 지정자를 자리표시자로 쓴다.

printf("Hello, CSGP\n");           /* 그대로 출력 */
printf("자본금 %d만원\n", money);   /* %d 자리에 money 값이 들어간다 */

%d 가 "여기에 정수를 넣어라"는 자리표시자이고, 뒤에 그 정수(money)를 준다. 서식 지정자는 자료형마다 다르다.

지정자 넣는 값 예
%d 정수 (int) printf("%d", 5000) → 5000
%s 문자열 printf("%s", "RPG") → RPG
%c 글자 (char) printf("%c", '#') → #
%f 실수 (double) printf("%f", 7.5) → 7.500000
%.1f 실수, 소수 1자리 printf("%.1f", 7.5) → 7.5

폭 지정도 알아 두면 표를 깔끔히 맞출 수 있다. % 와 지정자 사이에 숫자를 넣으면 최소 칸 수가 된다.

printf("%5d\n", 42);     /*  "   42"  — 오른쪽 정렬, 최소 5칸 */
printf("%-16s|\n", "EQMENT");  /* "EQMENT          |" — 왼쪽 정렬, 최소 16칸 */
printf("명성 ★ %d\n", fame);

현황판에서 회사명을 왼쪽으로 16칸 맞춰 찍고(%-16s), 월을 2칸으로 찍는(%2d) 이유가 이것이다. 서식으로 정렬하면 전각 문자 없이 ASCII만으로도 표가 반듯해진다.

2.2 이스케이프 문자 — 글자가 아닌 글자

\n 은 화면에 "역슬래시 n"으로 나오지 않고 줄바꿈이 된다. 이렇게 \ 로 시작하는 특수 표기를 이스케이프 문자라 한다.

표기 뜻
\n 줄바꿈 (다음 줄로)
\t 탭 (칸 맞추기)
\\ 역슬래시 \ 그 자체
\" 큰따옴표 " (문자열 안에서) 
printf("이름:\t김개발\n직업:\t개발자\n");
/* 이름:   김개발
   직업:   개발자   */

2.3 scanf — 입력받기

플레이어에게 값을 받을 땐 scanf 를 쓴다. 정수 하나를 받아 변수에 넣는 기본형은 이렇다.

int start;
printf("시작 자본을 입력하세요: ");
scanf("%d", &start);      /* 입력한 정수를 start 에 저장 */

주의할 점 하나 — scanf 의 변수 앞에는 &(앰퍼샌드) 를 붙인다. 이건 "이 변수가 사는 주소에 값을 써 넣어라"는 뜻이다. 왜 주소가 필요한지는 D 모듈(포인터)에서 완전히 밝혀진다. 지금은 "scanf 는 변수 앞에 &" 라는 규칙으로 외워 두면 된다. printf 에는 & 를 붙이지 않는다(보여 주기만 하니까).

⚠️ #define _CRT_SECURE_NO_WARNINGS. Visual Studio에서 scanf 를 쓰면 "안전하지 않다"는 경고(C4996)가 뜬다. 마이크로소프트가 자사 전용 scanf_s 를 권하기 때문인데, 우리는 표준 C 를 배우므로 scanf 를 그대로 쓰되 경고만 끈다. 소스 파일 맨 첫 줄에 이 한 줄을 두면 된다.

#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

2.4 게임에 적용 — 회사 현황판 (v0.1)

이제 1~2장을 합쳐 게임의 첫 화면을 만든다. 변수를 선언하고, 시작 자본을 입력받고, 현황판을 그린다.

#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

int main(void)
{
    /* --- 회사 상태 (아직 구조체는 없다. 낱개 변수로. 구조체는 9장) --- */
    int money = 5000;     /* 자본금 (만원) */
    int fame  = 0;        /* 명성 */
    int year  = 2026;     /* 연도 */
    int month = 1;        /* 월 */

    /* --- 시작 자본 입력 (원한다면 기본 5000 대신 직접) --- */
    printf("시작 자본을 입력하세요 (만원): ");
    scanf("%d", &money);

    /* --- 현황판 출력 --- */
    printf("==================================================\n");
    printf("  %-16s   %d년 %2d월     명성 ★ %d\n",
           "EQMENT STUDIO", year, month, fame);
    printf("  자본금 %d만원\n", money);
    printf("==================================================\n");
    return 0;
}

실행하면 이런 화면이 뜬다.

시작 자본을 입력하세요 (만원): 5000
==================================================
  EQMENT STUDIO      2026년  1월     명성 ★ 0
  자본금 5000만원
==================================================

%-16s 로 회사명을 왼쪽 16칸에 맞추고, %2d 로 월을 2칸에 맞춰 정렬이 반듯하다. 테두리는 전부 ASCII(=) 로 그렸다 — 콘솔 정렬·인코딩 문제를 애초에 없애기 위한 1부의 원칙이다(전각 테두리는 2부의 몫).

자본금의 콤마(5,000)는 왜 없나? 완성본 화면은 5,000만원 처럼 3자리마다 콤마를 찍는다. 하지만 콤마를 넣으려면 숫자를 뒤에서부터 세 자리씩 끊는 반복문과, 그걸 담아 두는 함수가 필요하다 — 둘 다 B 모듈에서 배운다. 그래서 B 모듈에서 MoneyStr 라는 함수로 이 화면을 한 단계 예쁘게 다듬을 것이다. 지금은 콤마 없이도 충분하다. "보이는 게 조금 투박해도, 배운 문법만으로 정직하게 만든다" — 이게 이 책의 방식이다.

3장 · 연산자 — 한 달을 정산하다

값을 담고 보여 줬으니, 이제 계산한다. 이 장의 게임 기능은 월 정산 — 한 달 동안 번 돈에서 세금과 임대료를 빼 순이익을 구하고, 자본금을 갱신하는 것이다(v0.2의 절반).

3.1 산술 연산자와 정수 나눗셈·나머지

기본 산술은 익숙하다. + - * /. 여기에 C만의 중요한 둘이 더 있다.

int a = 17, b = 5;
printf("%d\n", a + b);   /* 22 */
printf("%d\n", a - b);   /* 12 */
printf("%d\n", a * b);   /* 85 */
printf("%d\n", a / b);   /* 3   — 정수 나눗셈! 3.4 가 아니라 소수점 버림 */
printf("%d\n", a % b);   /* 2   — 나머지 (17 = 5*3 + 2) */
  • / (정수 나눗셈): 정수끼리 나누면 몫만 남고 소수점 이하는 버려진다(1.4에서 본 그 성질). 17 / 5 = 3.
  • % (나머지): 나누고 남은 나머지. 17 % 5 = 2.

이 둘은 게임에서 놀랄 만큼 자주 쓰인다. 판매 이익 "판매량 ÷ 2"(§GDD 5.6), 세금 "수입의 10%", 게이지를 몇 칸 채울지, 거스름돈 동전 개수 — 전부 / 와 % 다. 방금 1.4에서 "정수끼리 나누면 소수점이 잘린다"를 함정으로 배웠는데, 세금 계산에서는 이 잘림이 오히려 정확히 우리가 원하는 동작이다.

int income = 820;                  /* 이번 달 수입 (만원) */
int tax    = income * 10 / 100;    /* 세금 10% = 82. 소수점은 자연히 버려진다 */

income * 10 / 100 은 왜 income / 100 * 10 이 아닌가? 정수 나눗셈은 소수점을 버리므로 나누기를 나중에 해야 오차가 적다. 820 / 100 을 먼저 하면 8(0.2 손실)이 되어 8 * 10 = 80. 하지만 820 * 10 / 100 = 8200 / 100 = 82. 곱하기를 먼저, 나누기를 나중에 — 정수 연산의 기본 요령이다.

3.2 증감·복합 대입 — 자본금을 갱신하다

값을 1 늘리거나, 자기 자신에 무언가를 더해 다시 담는 일은 매우 흔해서 짧은 표기가 있다.

month++;          /* month = month + 1  (증가) */
money += profit;  /* money = money + profit  (복합 대입) */
money -= RENT;    /* money = money - RENT */

월 정산의 마지막은 "순이익을 자본금에 더하기"이므로 money += profit; 한 줄이다. 다음 달로 넘어갈 땐 month++;. 이 짧은 표기들이 앞으로 게임 코드 곳곳에 등장한다.

3.3 비교·논리 연산자 — 참과 거짓

계산 결과로 길을 가르려면 먼저 조건을 참/거짓으로 판별해야 한다. 그 판별을 하는 것이 비교·논리 연산자다. C에서 참은 1(0이 아닌 값), 거짓은 0 이다.

money < 0        /* 자본금이 음수인가?  (파산 판정) */
rating >= 9.0    /* 평점이 9.0 이상인가?  (명작 판정) */
sel == 6         /* 고른 메뉴가 6번인가?  ( == 는 '같다', = 는 '대입'! ) */
sel != 0         /* 0번이 아닌가? */

여러 조건을 엮을 땐 논리 연산자를 쓴다.

연산자 뜻 참이 되는 때
&& 그리고 (AND) 양쪽이 모두 참
|| 또는 (OR) 하나라도 참
! 아니다 (NOT) 뒤 조건이 거짓일 때 
if (money >= 10000 || rating >= 9.0)   /* 자산 1억 '또는' 명작이면 승리 */
    printf("승리!\n");

== 와 = 를 혼동하지 말 것. = 는 대입(상자에 넣기), == 는 비교(같은지 묻기)다. if (sel = 6) 이라고 쓰면 sel에 6을 대입해 버리고 항상 참이 되는 고약한 버그가 생긴다. 조건에서 "같은가?"는 반드시 == 두 개다.

3.4 연산자 우선순위

한 수식에 여러 연산자가 섞이면, 정해진 우선순위대로 계산된다. 곱셈·나눗셈이 덧셈·뺄셈보다 먼저인 건 수학과 같다.

int profit = income - tax - RENT;        /* 왼쪽에서 순서대로 빼기 */
int a = 2 + 3 * 4;                       /* 14 — 곱셈 먼저 (2 + 12) */
int b = (2 + 3) * 4;                     /* 20 — 괄호로 순서를 바꿈 */

헷갈리면 괄호로 의도를 분명히 하는 게 언제나 옳다. 컴파일러가 아니라 다음에 이 코드를 읽을 사람(대개 미래의 나)을 위한 배려다.

3.5 게임에 적용 — 월 정산 계산

3장을 모아 한 달을 정산한다. 이번 달 수입에서 세금과 임대료를 빼 순이익을 구하고, 자본금을 갱신한 뒤, GDD의 월 리포트(§3.3) 모양으로 출력한다. (아직 직원·판매 시스템이 없으니 수입은 예시로 입력받는다. 판매 수익은 D 모듈에서 진짜로 계산된다.)

const int RENT = 100;              /* 임대료 100만원/월 */

int income;
printf("이번 달 수입(만원): ");
scanf("%d", &income);

int tax    = income * 10 / 100;    /* 세금 10% (정수 나눗셈) */
int profit = income - tax - RENT;  /* 순이익 = 수입 - 세금 - 임대료 */
int before = money;
money += profit;                   /* 자본금 갱신 */

printf("============ %d년 %d월 결산 ============\n", year, month);
printf("  수입                     + %5d만원\n", income);
printf("  세금(10%%)                - %5d만원\n", tax);
printf("  사무실 임대료            - %5d만원\n", RENT);
printf("  --------------------------------------\n");
printf("  순이익                   %+6d만원\n", profit);
printf("  자본금  %d -> %d만원\n", before, money);
printf("========================================\n");

몇 가지 서식 요령이 보인다.

  • %% — printf 문자열 안에서 % 글자 자체를 찍으려면 %% 로 쓴다(10%% → 10%).
  • %+6d — + 를 붙이면 양수에도 부호를 표시한다(+570). 순이익의 흑자/적자를 부호로 한눈에.
  • %5d — 최소 5칸 오른쪽 정렬로 숫자 줄을 맞춘다.

실행하면 이렇게 나온다.

이번 달 수입(만원): 820
============ 2026년 1월 결산 ============
  수입                     +   820만원
  세금(10%)                -    82만원
  사무실 임대료            -   100만원
  --------------------------------------
  순이익                   +  638만원
  자본금  5000 -> 5638만원
========================================

4장 · 조건문 — 갈림길을 만들다

값을 담고(1장) 보여 주고 받고(2장) 계산했다(3장). 마지막은 갈라짐이다. 계산 결과에 따라 프로그램이 다른 길을 가게 만든다. 이 장의 게임 기능은 메인 메뉴 분기와 파산 판정이다(v0.2 완성).

4.1 if / else if / else

조건이 참이면 이 블록, 아니면 저 블록. 조건문의 기본이다.

if (money < 0) {
    printf("파산했습니다...\n");        /* 자본금이 음수면 */
} else if (money >= 10000) {
    printf("자산 1억 달성! 거상 엔딩\n"); /* 아니면서 1억 이상이면 */
} else {
    printf("회사를 계속 운영합니다.\n");  /* 그 밖의 경우 */
}

위에서부터 조건을 차례로 검사하다가, 처음 참이 되는 블록 하나만 실행하고 나머지는 건너뛴다. 그래서 순서가 중요하다 — 파산(가장 나쁜 경우)을 맨 위에서 먼저 거른다.

4.2 switch — 여러 갈래를 깔끔하게

메뉴처럼 "값이 1이면 A, 2면 B, 3이면 C…"로 여러 갈래가 필요할 땐 switch 가 if 사슬보다 읽기 좋다.

switch (sel) {
case 1:
    printf("게임 개발 화면으로\n");
    break;                    /* ← break 를 빼먹으면 아래로 줄줄이 실행된다! */
case 2:
    printf("직원 관리 화면으로\n");
    break;
case 0:
    printf("게임을 종료합니다\n");
    break;
default:
    printf("잘못된 선택입니다\n");   /* 어느 case 에도 안 맞으면 */
    break;
}

⚠️ break 를 잊지 말 것. switch 는 맞는 case 로 점프한 뒤, break 를 만날 때까지 아래 case 들을 계속 실행한다(이를 "폴스루"라 한다). case 1 뒤에 break 가 없으면 case 2 코드까지 이어서 실행되는 버그가 생긴다. 각 case 끝의 break 는 습관으로 챙기자.

4.3 삼항 연산자 — 한 줄짜리 선택

"조건이 참이면 A, 아니면 B"를 값 하나로 고를 때는 삼항 연산자 조건 ? A : B 가 간결하다.

const char *msg = (money < 0) ? "파산" : "정상";   /* 음수면 "파산", 아니면 "정상" */
int bonus       = (rating >= 8.0) ? 10 : 5;        /* 평점 8 이상이면 10, 아니면 5 */

if/else 로도 쓸 수 있지만, 값 하나를 고르는 단순한 경우엔 삼항이 짧고 깔끔하다. 남용하면 오히려 읽기 어려워지니, "한눈에 들어올 때만" 쓰는 게 요령이다. (이 삼항 연산자는 D 모듈의 실습 미션에서 공격 위력을 고를 때 다시 등장한다.)

4.4 게임에 적용 — 메뉴 분기와 파산 판정

이제 A 모듈 전체를 하나로 묶는다. 현황판을 그리고(2장), 정산하고(3장), 메뉴를 받아 갈라지고(switch), 마지막에 파산을 판정한다(if). 아직 while 이 없어 딱 한 바퀴만 돌지만, 이 한 바퀴가 곧 게임의 한 달이다.

#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

int main(void)
{
    const int RENT = 100;

    int money = 5000, fame = 0, year = 2026, month = 1;
    int income, sel;

    /* ── 현황판 (v0.1) ───────────────────────────── */
    printf("==================================================\n");
    printf("  %-16s   %d년 %2d월     명성 ★ %d\n",
           "EQMENT STUDIO", year, month, fame);
    printf("  자본금 %d만원\n", money);
    printf("==================================================\n");
    printf("  [1] 게임 개발     [2] 직원 관리\n");
    printf("  [6] 다음 달 진행  [0] 게임 종료\n");
    printf("  >> 선택: ");
    scanf("%d", &sel);

    /* ── 메뉴 분기 (switch) ──────────────────────── */
    switch (sel) {
    case 1:
        printf("아직 개발 시스템이 없습니다 (9장에서 추가).\n");
        break;
    case 2:
        printf("아직 직원 시스템이 없습니다 (7장에서 추가).\n");
        break;
    case 6:
        /* ── 다음 달 진행 = 월 정산 (v0.2) ── */
        printf("이번 달 수입(만원): ");
        scanf("%d", &income);
        {
            int tax    = income * 10 / 100;
            int profit = income - tax - RENT;
            money += profit;
            month++;                          /* 달력 넘김 */
            printf("순이익 %+d만원,  자본금 -> %d만원\n", profit, money);
        }
        break;
    case 0:
        printf("게임을 종료합니다.\n");
        break;
    default:
        printf("잘못된 선택입니다.\n");
        break;
    }

    /* ── 파산 판정 (if) ──────────────────────────── */
    if (money < 0)
        printf(">> 자본금이 바닥났습니다. 파산 엔딩!\n");

    return 0;
}

이 프로그램은 아직 "한 턴"만 돈다. case 6 으로 한 달을 진행하고 나면 프로그램이 끝난다. 여기에 "메뉴로 돌아가 계속" 을 붙이는 순간, 즉 이 전체를 while 로 감싸는 순간, 진짜 게임이 된다 — 그게 바로 다음 B 모듈이다.

A 모듈의 도착점. 우리는 지금 값(1장)·출력·입력(2장)·계산(3장)·분기(4장)를 모두 손에 넣었고, 그것으로 게임의 "한 달"을 만들었다. GDD의 마일스톤으로 치면 v0.2 — "1턴짜리 게임"이다. B 모듈에서 이 한 달을 반복문으로 감싸면 v0.3 "돌아가는 회사"가 된다.

코드 워크스루 — 실제 완성본에서 보는 값·계산·분기

지금 우리가 낱개 변수로 만든 화면이, Study/Tycoon/ 의 완성본(v1.0)에서는 어떤 모습인지 확인하자. 완성본은 구조체(Company, 9장)를 쓰지만, 찍히는 값과 계산은 우리가 A 모듈에서 배운 그대로다.

실제 파일: Study/Tycoon/ui.c — 현황판 출력과 서식 함수

① 현황판 헤더 (ui.c · PrintHeader) — 완성본은 회사 상태를 Company *c 구조체로 받지만, 실제로 하는 일은 우리 2.4의 printf 와 똑같다. %-16s(회사명 왼쪽 정렬), %2d(월 2칸)로 표를 맞춘다.

void PrintHeader(const Company *c)
{
    char m[24];
    MoneyStr(c->money, m, sizeof(m));    /* 자본금에 3자리 콤마 (B모듈에서 배울 함수) */
    printf("==================================================\n");
    printf("  %-16s   %d년 %2d월     명성 ★ %d\n",
           c->name, c->year, c->month, c->fame);
    printf("  자본금 %s만원 | 직원 %d명 | 개발중 %d건 ...\n",
           m, c->staffCount, c->proj.active);
    printf("==================================================\n");
}

여기서 c->money, c->year 는 "회사 c 안의 자본금·연도"를 꺼내는 문법(D 모듈)일 뿐, 그 값을 %d 로 찍는 부분은 A 모듈 그대로다. 우리가 지금 배운 것이 완성본에서도 그대로 쓰인다 — 낱개 변수 money 가 나중에 c->money 로 이름만 바뀔 뿐이다.

② 3자리 콤마 (ui.c · MoneyStr) — 자본금 5638 을 5,638 로 바꾸는 함수. 자릿수를 뒤에서부터 세며 3자리마다 콤마를 끼운다. 정수 나눗셈·나머지·반복문·함수가 모두 들어간, A~B 모듈 문법의 종합 응용이다.

if (i > 0 && (n - i) % 3 == 0)   /* 남은 자릿수가 3의 배수인 자리 앞에 콤마 */
    out[o++] = ',';

(n - i) % 3 == 0 — 3장에서 배운 나머지 연산자 % 가 "3자리마다"라는 규칙을 정확히 표현한다. 이 함수의 전체 구현(반복문·함수·문자열 배열)은 B·C 모듈을 지나야 온전히 읽히지만, 그 심장인 % 는 지금 이미 손에 있다.

③ 게이지 (ui.c · DrawGauge) — 진행도 [####------] 를 그리는 함수도 val * width / maxv(비율을 칸 수로) 라는 곱하기 먼저, 나누기 나중 의 정수 연산이다(3.1의 요령 그대로).

이 세 조각을 관통하는 사실 하나 — 화면에 보이는 모든 숫자는 결국 변수(1장)에 담겨, 연산자(3장)로 계산되고, 서식(2장)으로 찍히며, 조건(4장)으로 갈라진다. 완성본이 아무리 커도, 그 바닥은 이 모듈에서 배운 네 가지다.

그림으로 정리 — 값이 흐르는 길

📊 웹 교육문서에서는 이 정산 흐름이 단계별 애니메이션(V-P1-A)으로 재생된다. 수입에서 세금·임대료가 빠져 순이익이 되고, 자본금이 갱신되는 과정을 값의 이동으로 보여 준다. 책에서는 아래 도식으로 대신한다.

한 달 정산의 흐름 — 값이 변수 상자를 옮겨 다닌다.

  입력                계산 (연산자)                    갱신·판정
  ┌──────────┐      income * 10 / 100 = tax           money += profit
  │ income   │──┐   income - tax - RENT = profit      ┌───────────────┐
  │  820     │  │        820 - 82 - 100 = 638         │ money 5000    │
  └──────────┘  └──────────────────────────────────▶ │   → 5638      │
   const RENT 100                                     └───────┬───────┘
   세율 10%                                                   │  if (money < 0)?
                                                              ▼
                                                     거짓 → 회사 유지
                                                     참   → 파산 엔딩

자료형이라는 상자의 모양

  int money  = 5000;    ┌──────────┐   정수만.  5000, -100, 0
                        │  5000    │   소수점을 못 담는다 → 7/2 = 3
                        └──────────┘

  double rating = 7.5;  ┌──────────┐   소수 가능.  7.5, 0.0
                        │  7.5     │   평점처럼 '나눈 값'을 담을 때
                        └──────────┘

  char  grade = 'A';    ┌───┐         글자 하나.  'A', '#', '\n'
                        │ A │
                        └───┘

한 문장으로: 자료형은 값을 담는 상자의 모양이고, 연산자는 그 값을 바꾸는 손이며, 조건문은 값을 보고 길을 고르는 눈이다.

체크포인트 — 스스로 점검하기

각 문항을 먼저 스스로 답해 본 뒤, 아래 해설을 펼쳐 확인하자.

Q1. double r = 7 / 2; 를 실행하면 r 은 3.5 가 아니라 3.0 이다. 왜인가? 3.5 를 얻으려면 어떻게 고치나?

Q2. printf("세율 %d%%\n", 10); 의 출력은? %% 가 필요한 이유는?

Q3. income = 950 일 때 tax = income * 10 / 100; 과 tax = income / 100 * 10; 의 결과가 다르다. 각각 얼마이고, 왜 다른가?

Q4. switch 의 case 1: 블록 끝에 break; 를 빼먹으면 무슨 일이 일어나는가?

Q5. 아래 코드는 자본금이 얼마든 항상 "파산"을 출력한다. 왜이고, 어떻게 고치나?

if (money = 0)
    printf("파산\n");
해설 펼치기

A1. 7 과 2 가 둘 다 정수라 7 / 2 가 정수 나눗셈으로 먼저 계산되어 3(소수점 버림)이 되고, 그 3을 double 에 담아 3.0 이 된다. double r = 7.0 / 2; 처럼 한쪽을 실수로 만들면 실수 나눗셈이 되어 3.5 가 된다. 변수라면 (double)a / b 로 캐스트한다.

A2. 세율 10%. printf 는 % 를 서식 지정자의 시작으로 해석하므로, % 글자 자체를 찍으려면 %% 로 이스케이프해야 한다. %d%% = "정수 하나 + 퍼센트 기호".

A3. income * 10 / 100 = 9500 / 100 = 95. income / 100 * 10 = 9(→9.5의 버림)... 9 * 10 = 90. 정수 나눗셈은 소수점을 버리므로, 나누기를 먼저 하면 950/100=9 에서 0.5가 사라져 오차가 커진다. 곱하기를 먼저, 나누기를 나중에 해야 정확하다.

A4. case 1: 을 실행한 뒤 break 를 못 만나 바로 아래 case 2: 코드까지 이어서 실행된다(폴스루). 의도치 않게 두 갈래가 다 실행되는 버그다. 각 case 끝에는 break; 를 챙긴다.

A5. if (money = 0) 는 ==(비교)가 아니라 =(대입)이다. money에 0을 대입한 뒤 그 결과값 0을 조건으로 쓰는데… 잠깐, 0은 거짓이니 이 경우 오히려 출력이 안 된다. 진짜 함정은 if (money = 5) 처럼 0이 아닌 값을 대입할 때로, 그러면 항상 참이 된다. 어느 쪽이든 의도는 비교이므로 if (money == 0) 이 맞다. = 와 == 를 혼동하면 "대입해 버리고 그 값으로 분기하는" 고약한 버그가 된다. (money < 0 파산 판정도 마찬가지로 비교 연산자다.)

실습 미션 — 가위바위보 심판 rps.c ⭐

본편(타이쿤)이 "한 게임을 계속 키우는 세로 축"이라면, 실습 미션은 이 모듈의 새 문법만 콕 집어 연습하는 독립 미니게임(가로 축) 이다. 먼저 직접 짜 보고, 막히면 힌트를, 그래도 막히면 정답을 펼쳐 대조하자.

드릴 문법 — int 변수 · scanf/printf · 비교·논리 연산자 · 정수 나머지(%) · if/switch/삼항 만드는 것 — 두 사람의 가위바위보 한 판을 입력받아 승패를 판정하는 심판 프로그램. 클리어 조건 — 두 손을 숫자로 입력받아, 무승부/1P 승/2P 승을 정확히 가려 출력한다. 규모 — 약 40줄, 파일 1개. 표준 C만(printf/scanf). 아직 반복문·난수는 쓰지 않는다(한 판만, 랜덤은 E 모듈).

문제 정의

가위바위보의 규칙은 순환한다 — 가위는 보를, 바위는 가위를, 보는 바위를 이긴다. 이 순환을 조건문으로 곧이곧대로 나열해도 되고(직관적), 3장에서 배운 나머지 연산자 % 로 우아하게 한 방에 판정할 수도 있다(이 미션의 숨은 재미). 두 방식을 모두 경험하는 게 목표다.

입력은 1=가위, 2=바위, 3=보 로 받는다.

목표 / 클리어 조건

  1. int 변수 두 개에 두 사람의 손(1~3)을 scanf 로 입력받는다.
  2. 잘못된 입력(1~3 밖)이면 안내하고 종료한다 — if 로 범위 검증.
  3. 무승부 / 1P 승 / 2P 승을 판정해 출력한다.
  4. 각자가 낸 손을 이름("가위"/"바위"/"보")으로도 보여 준다 — switch 또는 문자열 배열.

힌트

힌트 1 — 승패를 조건문으로 (직관적 방법)

무승부를 먼저 거르고, 1P가 이기는 세 경우를 || 로 묶는다.

if (a == b)
    puts("무승부");
else if ((a==1 && b==3) || (a==2 && b==1) || (a==3 && b==2))
    puts("1P 승!");     /* 가위>보, 바위>가위, 보>바위 */
else
    puts("2P 승!");

(puts 는 문자열 하나를 줄바꿈까지 붙여 출력하는 간편 함수다.)

힌트 2 — 나머지 연산자 한 방 (우아한 방법)

손을 0=가위, 1=바위, 2=보 로 바꾸면(입력값 −1), 이 순환에는 예쁜 규칙이 숨어 있다.

int x = a - 1, y = b - 1;          /* 1~3 → 0~2 */
int r = (x - y + 3) % 3;           /* 0=무승부, 1=1P승, 2=2P승 */

(x - y + 3) % 3 이 왜 되는지는 정답 코드의 주석에서 설명한다. 나머지 % 로 순환 규칙을 단 한 줄에 담는 이 기법을 눈에 익혀 두자.

힌트 3 — 손 이름 출력

switch (a) 로 1→"가위" 2→"바위" 3→"보"를 출력해도 되고, const char *NAME[] = {"가위","바위","보"}; 배열을 만들어 NAME[a-1] 로 꺼내도 된다(배열은 C 모듈에서 정식으로 배우지만, 미리 맛봐도 좋다).

정답 코드

정답 펼치기 — rps.c 
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

int main(void)
{
    const char *NAME[3] = { "가위", "바위", "보" };  /* 0,1,2 순서 */
    int a, b;

    printf("=== 가위바위보 심판 (1:가위 2:바위 3:보) ===\n");
    printf("1P 손: ");  scanf("%d", &a);
    printf("2P 손: ");  scanf("%d", &b);

    /* 범위 검증 — 조건문으로 잘못된 입력 거르기 */
    if (a < 1 || a > 3 || b < 1 || b > 3) {
        printf("! 1~3 사이 숫자만 입력하세요.\n");
        return 1;
    }

    printf("1P [%s]  vs  2P [%s]\n", NAME[a - 1], NAME[b - 1]);

    /* 나머지 연산자로 순환 승패를 한 방에 판정.
       0=가위,1=바위,2=보 로 두면, x가 y를 이기는 경우는
       항상 (x - y + 3) % 3 == 1 이 된다.
       가위(0)>보(2): (0-2+3)%3 = 1
       바위(1)>가위(0): (1-0+3)%3 = 1
       보(2)>바위(1):  (2-1+3)%3 = 1   */
    int x = a - 1, y = b - 1;
    int r = (x - y + 3) % 3;   /* 0=무승부, 1=1P승, 2=2P승 */

    switch (r) {
    case 0: printf(">> 무승부!\n");            break;
    case 1: printf(">> 1P 승리!\n");           break;
    case 2: printf(">> 2P 승리!\n");           break;
    }
    return 0;
}

빌드: cl rps.c (MSVC) 또는 gcc rps.c -o rps

함정/체크포인트

  • if (a < 1 || a > 3 …) 의 논리 연산자 || — 하나라도 범위를 벗어나면 거른다. && 로 바꾸면 "네 조건이 전부 참일 때만" 걸러서 사실상 검증이 안 된다. AND/OR을 정확히 골라야 한다.
  • (x - y + 3) % 3 에서 +3 이 왜 필요한가? x - y 가 음수(예: 0 - 2 = -2)일 수 있는데, C의 % 는 음수에서 우리가 원하는 0~2를 안 준다. +3 을 더해 항상 양수로 만든 뒤 % 3 하면 깔끔히 0~2로 접힌다. 나머지 연산을 순환에 쓸 때의 상투적 요령이다.
  • 직관적 방법(힌트 1)과 나머지 방법(정답)의 결과가 같은지 두 방식으로 다 짜 보면, % 가 조건문 여섯 줄을 한 줄로 줄이는 힘을 실감할 수 있다.

확장 과제

  • B 모듈(반복문) 에서 while 로 감싸 "3판 2선승"으로 키운다.
  • E 모듈(난수) 에서 2P를 컴퓨터로 바꿔 rand() % 3 으로 손을 내게 한다.

이 모듈을 마치며

얻은 것 다음에 쓰는 곳
변수·자료형으로 상태 담기 모든 후속 장 — 게임의 모든 값이 변수에서 시작
printf 서식으로 화면 정렬 B 모듈 함수 분리, 전 장의 화면 출력
정수 나눗셈·% 로 계산 세금·판매 이익·게이지·평점(D 모듈)
switch/if 로 분기 B 모듈 월 루프, 4·D 모듈 메뉴, 파산·승리 판정

우리는 프로그래밍의 세 기둥 — 값·계산·분기 — 를 세웠고, 그것으로 게임의 "한 달"을 만들었다. 아직 한 번밖에 못 돌지만, 그 한 번 안에 게임의 심장 박동이 다 들어 있다. 다음 B 모듈에서 이 한 달을 반복문으로 감싸는 순간, 우리 회사는 비로소 살아 움직이기 시작한다.
