심화 1

영웅은 절차적 · HeroIsProcedural

01. HeroIsProcedural.cpp

주제 — 클래스 없이 지역변수와 함수만으로 텍스트 RPG를 완성 (제목 그대로 "절차적").

  • #define으로 가위(GA)·바위(BA)·보(BO) 상수 정의
  • 몬스터/서브보스/보스 능력치를 다수 변수로 관리, game_level로 스케일링
  • 게임 루프 while(1) + 메뉴 분기(스테이터스/탐험/상점/저장)
  • 전투 = 가위바위보 승패 판정, HP·경험치·골드·레벨업 로직(while(exp > up_exp))
  • 함수 분리: GameContent(name, level)로 실제 게임 실행
💡정리 — Basic01~03 문법만으로도 게임이 만들어짐을 체험. 동시에 변수가 너무 많아 관리가 어렵다는 절차적 방식의 한계를 느끼는 단계.
심화 2

탈출 루프 · DirectRope

02. DirectRope.cpp

주제 — 2차원 배열 맵 위에서 캐릭터 이동.

  • int map[SIZE][SIZE] 타일맵, 값(0/1/2)에 따라 벽·플레이어·탈출구 렌더링

심화 3

미로 탈출 · EscapeMirror

03. EscapeMirror.cpp

주제 — 구조체를 본격 활용한 미로/채굴 게임.

  • 여러 구조체 협업: PLAYER, ITEM, TILE, KEY
  • TILE[SIZE][SIZE] 2차원 구조체 배열로 지형·아이템·열쇠·상점·탈출구를 한 칸에 표현
  • 상태 머신: stage 값(1 초기화 / 2 타이틀 / 3 게임오버 / 4 상점 / 그 외 플레이)으로 화면 전환

  • 스태미나·골드·채굴·열쇠 수집 → 모든 열쇠를 모아야 탈출, 스테이지 진행
  • 로딩/애니메이션 프레임 연출
💡정리 — 구조체로 게임 오브젝트를 모델링하고, 상태머신으로 흐름을 제어하는 설계 감각.
심화 4

겜블 게임 · Gamble

01. Gamble.cpp · GameMain.* · USER.* · DEAL.* · config.h

주제 — 절차적 → 객체지향으로 넘어가는 전환점. 처음으로 class 도입.

    · GameMain — 게임 전체 진행(그리기/입력/베팅/AI/승패)

심화 5 · 최종

알카노이드 · Alcanoid

Alcanoid.* · GameObject.* · Actor.* · Ball.* · Block.* · GameManager.*

벽돌깨기 게임. 객체지향의 4대 요소를 모두 실습하는 커리큘럼의 정점.

  • 상속(Inheritance): GameObject(위치·방향·속도·모양·생존 등 공통) → 이를 상속한 Actor(패들)·Ball(공)·Block(벽돌)
  • 캡슐화(Encapsulation): protected 멤버 + Get/Set 접근자
  • 다형성(Polymorphism): Collide(GameObject*, GameObject*) 부모 포인터로 자식 통일 처리, DrawObject() 오버로딩
  • 게임 매니저 패턴: GameManager가 표준 게임 루프를 주도 —
    SetScreen → Initialize → (GetKeyEvent → InputProcess → Update → DrawScreen → FrameDelay) 반복
  • enum SCENE(START/PLAY/OVER), enum BLOCKNUM(STAGE1~3)로 상태·스테이지 표현
  • 동적 배열 new Block[블록수] / delete[], 스테이지별 블록 수 증가
  • 충돌 판정(공↔패들, 공↔벽돌, 공↔벽)과 반사, 삼각함수 각도 계산(cos/sin)
  • 프레임 제어: clock()으로 경과 시간 측정 → GAME_FPS에 맞춰 Sleep
  • WinAPI 콘솔 제어: COORD + SetConsoleCursorPosition, WriteFile 직접 출력(깜빡임 최소화)
  • 파일 입출력: ofstream/ifstream으로 save_point.txt에 스테이지·점수·블록 상태 저장/불러오기(S/L 키)
🏆정리 — 문법 → 절차적 게임 → OOP 설계까지, 배운 모든 것을 하나의 완성된 게임으로 통합.
Summary

핵심 개념 총정리

강의를 통해 다진 개념을 계층별로 요약하면:

①언어 기초

  • 자료형과 크기(sizeof, <limits.h>), 형변환 캐스팅
  • 연산자: 산술 · 비교 · 논리 · 나머지 % · 삼항 ?:
  • 제어 흐름: if/else if/switch, for/while, 중첩 반복

②데이터 구조화

  • 배열(1D/2D), 구조체(struct), 구조체 배열
  • 아스키코드와 문자 처리

③함수와 메모리

  • 함수 정의/반환/매개변수, 함수 오버로딩
  • 포인터(*, &, NULL), 참조(&), 전역변수
  • 동적 할당 new/delete[]
  • 템플릿 template<typename T>

④객체지향(OOP)

  • 클래스, 캡슐화(getter/setter, 접근 지정자)
  • 상속(: public GameObject), 다형성(부모 포인터·오버로딩)
  • 헤더/구현 분리, 객체 생명주기(생성자/소멸자)

⑤게임 · 시스템

  • 게임 루프와 상태 머신, 프레임 제어(clock)
  • 콘솔 실시간 입력(_kbhit/_getch), WinAPI 화면 제어
  • 파일 입출력, 충돌 판정, 랜덤(srand/rand)
Design Intent

학습 흐름의 설계 의도

이 커리큘럼은 "문법을 배운다 → 문법으로 게임을 만든다 → 게임이 커지면서 더 나은 설계(OOP)가 필요해진다"는 흐름으로 짜여 있다.

Basic01~04에서 재료(문법)를 모으고,

HeroIsProcedural에서 재료만으로 게임을 만들며 절차적 방식의 한계(변수 폭발)를 느끼고,

Gamble에서 클래스로 정리하는 맛을 보고,

Alcanoid에서 상속·다형성으로 확장 가능한 구조를 완성한다.

즉, "왜 객체지향이 필요한가"를 직접 겪으면서 체득하도록 설계된 실전형 커리큘럼이다.
