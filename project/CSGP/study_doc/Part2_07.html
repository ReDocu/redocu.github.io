6단계 · Maze — 알고리즘 심화 (생성 + 탐색)

단계 6 ⭐ · 게임 Algo Maze · 씬 클래스 MazeContent : public IGameContent 처음 배우는 기술 — 격자 좌표계(2·셀+1) 미로 생성 5종 Binary Tree(편향) Recursive Backtracker(스택·DFS) Randomized Prim(프론티어) Kruskal + union-find(분리집합·MST) Recursive Division(분할정복) 경로탐색 4종(BFS·DFS·A*·벽 따라가기) BFS 최단경로 복원 예상 학습 시간 — 6–8시간

Tetris에서 "자료구조 위의 알고리즘"을 맛봤다면, Maze는 알고리즘을 주제 그 자체로 끌어올린 심화 단계다. 이 게임은 스테이지마다 다른 알고리즘으로 미로를 짓는다 — 그래서 미로의 생김새가 스테이지마다 확연히 달라진다. 편향된 대각선, 구불구불한 강줄기, 잔가지 덤불, 균일한 격자, 방과 복도. 그리고 어느 미로에서든 BFS·DFS·A·벽 따라가기*가 길을 찾는 과정을 눈으로 본다. 생성 5종 + 탐색 4종, 아홉 가지 알고리즘을 한 게임에서 보고 겪는다. 여기서 배우는 절차적 생성과 경로탐색은 곧바로 마지막 캡스톤 Battle City에서 다시 쓰인다.

이 단계의 학습 목표

이 단계를 마치면 다음을 할 수 있다.

  1. (2·셀+1) 격자 모델로 미로를 표현하고, 셀 좌표와 벽 좌표를 구분할 수 있다.
  2. 다섯 가지 생성 알고리즘의 아이디어·자료구조·시그니처(생김새)를 설명할 수 있다.
  3. union-find(분리집합) 로 사이클을 피하며 미로를 만드는 Kruskal을 이해한다.
  4. 분할정복으로 미로를 재귀적으로 나누는 Recursive Division을 짤 수 있다.
  5. BFS·DFS·A·벽 따라가기*의 차이를 알고, 왜 BFS가 무가중 격자에서 최단인지 설명할 수 있다.
  6. BFS의 prev 배열로 최단경로를 복원하고, 걸음 효율로 채점할 수 있다.

왜 Maze가 이 자리인가

Snake~Galaga까지, 우리는 게임을 만드는 기법을 배웠다 — 자료구조, 물리, 시스템, 성능. Maze의 임무는 다르다. 게임은 이제 알고리즘을 가르치는 무대가 된다.

핵심 통찰 하나 — "미로를 어떻게 만드느냐"에 따라 미로의 성격이 완전히 달라진다. 같은 격자, 같은 규칙(완전연결 미로)인데도, Binary Tree로 만들면 대각선으로 편향되고, DFS로 만들면 강줄기처럼 구불거리고, Kruskal로 만들면 균일하게 흩어진다. 알고리즘이 곧 미로의 얼굴이다. 그래서 이 게임은 스테이지를 생성 알고리즘으로 나눈다 — 스테이지가 바뀌면 미로의 생김새가 통째로 바뀐다.

그 위에 두 번째 알고리즘 계열 — 경로탐색 — 을 얹는다. 어느 미로에서든 "이 미로를 어떻게 푸느냐"를 BFS·DFS·A*·벽 따라가기로 시각화해 비교한다. BFS는 물결처럼 균등하게 퍼지고, A*는 출구 쪽으로 편향돼 적게 탐색하며, DFS는 한 방향으로 파고들다 헤매고, 벽 따라가기는 오른손을 벽에 대고 우회한다. 같은 문제를 푸는 네 가지 태도를 눈으로 본다.

  • 생성 알고리즘 — Binary Tree(편향) → DFS(스택·백트래킹) → Prim(프론티어) → Kruskal(union-find·MST) → Division(분할정복).
  • 탐색 알고리즘 — BFS(큐·최단) · DFS(스택) · A*(휴리스틱) · 벽 따라가기(무메모리).

커리큘럼의 수렴 지점. 여기서 익힌 절차적 생성은 마지막 캡스톤 Battle City의 전장 생성으로, BFS 경로탐색은 그 적 탱크의 추적 AI로 곧바로 재사용된다. Maze를 캡스톤 직전에 두는 이유다 — "배우고 즉시 응용". 그리고 1부의 씨앗들이 여기서 열매를 맺는다: E 모듈의 Fisher-Yates 셔플이 Kruskal에, 재귀·스택 개념이 Backtracker와 Division에.

개념 1 · 격자 모델 — 셀과 벽

미로를 배열로 표현하는 방법이 핵심이다. 방(셀) 사이에 벽이 놓인다. 그래서 셀이 13×10이면, 벽까지 담는 격자는 그 2배+1 크기다.

static const int CELL_COLS = 13, CELL_ROWS = 10;
static const int GRID_W = CELL_COLS * 2 + 1;   // 27 (셀 + 사이 벽)
static const int GRID_H = CELL_ROWS * 2 + 1;   // 21

bool wall[GRID_H][GRID_W];   // true = 벽, false = 통로

규칙은 좌표의 홀짝으로 나뉜다.

  • 셀(방) — 홀수 좌표. 셀 (cx, cy) ↔︎ 격자 (2cx+1, 2cy+1). 항상 통로.
  • 벽 — 짝수 좌표. 두 인접 셀 사이에 놓인다.

미로 만들기는 이렇게 시작한다 — 모든 칸을 벽으로 채우고, 생성 알고리즘이 셀 사이 벽을 뚫어(carve) 통로를 낸다.

void ResetWallsAll(bool val) { /* 전부 벽으로 */ }
void OpenAllCells() {                       // 모든 셀(방)을 통로로 열기
    for (int cy = 0; cy < CELL_ROWS; cy++)
        for (int cx = 0; cx < CELL_COLS; cx++)
            wall[2*cy+1][2*cx+1] = false;    // 홀수 좌표 = 방
}

방향은 단위 벡터로 다룬다(0=북, 1=동, 2=남, 3=서). 이 DX/DY 는 생성(셀 이동)과 탐색(격자 이동) 양쪽에 쓰인다.

static const int DX[4] = { 0, 1, 0, -1 };   // 북 동 남 서
static const int DY[4] = { -1, 0, 1, 0 };

두 셀 (cx,cy) 와 방향 d 의 이웃 사이 벽을 뚫는 것은, 그 중간 격자칸을 false 로 만드는 것이다.

wall[2*cy + 1 + DY[d]][2*cx + 1 + DX[d]] = false;   // 셀과 이웃 사이 벽 제거

격자 좌표와 화면 좌표를 구분하라. 게임 로직은 wall[gy][gx] 격자 인덱스로만 계산하고, 그릴 때만 ((MAZE_X0+gx)*2, MAZE_Y0+gy) 로 변환한다(전각 x*2). 격자 인덱스와 콘솔 좌표를 혼동하지 않는 것 — Tetris·Road Fighter에서 계속 강조한 그 규율이다.

개념 2 · 생성 알고리즘 5종 — 스테이지가 곧 알고리즘

다섯 스테이지는 각각 하나의 생성 알고리즘이다. 같은 격자에서 시작하지만 결과의 얼굴이 전혀 다르다.

스테이지 알고리즘 자료구조 시그니처(생김새) 배우는 핵심
1 Binary Tree 없음 우상단 대각선 편향, 위 행·오른쪽 열이 뻥 뚫림 "편향(bias)"이란
2 Recursive Backtracker 스택+방문 길고 구불구불한 강줄기 통로 스택·백트래킹·DFS
3 Randomized Prim 프론티어 짧은 통로 + 잔가지 덤불 프론티어·그리디 성장
4 Kruskal union-find 편향 없이 균일 분리집합·MST·사이클 회피
5 Recursive Division 재귀 방과 좁은 문의 격자형 분할정복

① Binary Tree — 가장 단순한 생성. 모든 셀을 훑으며 각 셀에서 "북 또는 동" 중 하나를 랜덤으로 골라 벽을 뚫는다.

void MazeContent::GenBinaryTree()
{
    ResetWallsAll(true);  OpenAllCells();
    for (int cy = 0; cy < CELL_ROWS; cy++)
        for (int cx = 0; cx < CELL_COLS; cx++) {
            bool canN = (cy > 0), canE = (cx < CELL_COLS - 1);
            if (!canN && !canE) continue;
            bool carveEast = canE;
            if (canN && canE) carveEast = (rand() % 2 == 0);   // 둘 다 되면 동전 던지기
            if (carveEast) wall[2*cy+1][2*cx+2] = false;       // 동쪽 벽 뚫기
            else           wall[2*cy][2*cx+1]   = false;       // 북쪽 벽 뚫기
        }
}

이 단순함이 편향을 낳는다 — 맨 위 행의 셀은 "북"이 없어 항상 "동"만 뚫리고(위 행이 일자 통로), 맨 오른쪽 열은 항상 "북"만 뚫린다. 그래서 미로가 우상단으로 쏠린다. "가장 단순한 알고리즘은 가장 편향된 결과를 낸다" — 편향이 무엇인지 눈으로 배우는 튜토리얼 스테이지다.

② Recursive Backtracker (DFS) — 스택으로 파고들기. 한 셀에서 미방문 이웃으로 벽을 뚫고 전진하다(스택 push), 막다른 곳이면 되돌아온다(스택 pop). 깊은 재귀 대신 명시적 스택을 쓴다.

void MazeContent::GenBacktracker()
{
    ResetWallsAll(true);  OpenAllCells();
    bool visited[NUM_CELLS] = { false };
    std::vector<int> st;                       // 명시적 스택
    visited[0] = true;  st.push_back(0);

    while (!st.empty()) {
        int cur = st.back();
        int cx = cur % CELL_COLS, cy = cur / CELL_COLS;

        int cand[4], nc = 0;                   // 미방문 이웃 후보 수집
        for (int d = 0; d < 4; d++) {
            int nx = cx + DX[d], ny = cy + DY[d];
            if (nx<0||nx>=CELL_COLS||ny<0||ny>=CELL_ROWS) continue;
            if (!visited[ny*CELL_COLS + nx]) cand[nc++] = d;
        }
        if (nc == 0) { st.pop_back(); continue; }   // 막다름 → 백트래킹

        int d = cand[rand() % nc];             // 랜덤 이웃 선택
        int nx = cx + DX[d], ny = cy + DY[d];
        wall[2*cy+1+DY[d]][2*cx+1+DX[d]] = false;   // 벽 뚫고 전진
        visited[ny*CELL_COLS + nx] = true;
        st.push_back(ny*CELL_COLS + nx);
    }
}

한 방향으로 끝까지 파고들다 막히면 되돌아오는 이 깊이 우선(DFS) 성격이, 길고 구불구불한 강줄기 통로를 만든다. 스택이 "지금까지 온 길"을 기억하고, pop 이 "갈림길로 되돌아가기"다. 스택 = 되돌아갈 길의 기억 — 이 감각이 곧 탐색의 DFS로 이어진다.

③ Randomized Prim — 프론티어에서 자라기. 시작 셀을 미로에 편입하고, 그 인접 벽들을 프론티어(경계 후보) 에 넣는다. 프론티어에서 랜덤으로 하나 골라 미로에 연결하고, 새로 편입된 셀의 이웃을 다시 프론티어에 넣는다. 미로 안 셀에서 사방으로 조금씩 자라니, 짧은 통로와 잔가지가 많은 덤불형이 된다.

④ Kruskal + union-find, ⑤ Recursive Division 은 각각 개념 3·4에서 깊이 본다.

각 스테이지의 벽 색을 알고리즘마다 다르게 해(회색·남색·초록·보라·빨강), 시그니처를 색으로도 구분한다.

개념 3 · union-find — Kruskal 미로의 심장

Kruskal 미로는 아름답다. 모든 벽(간선)을 섞어 하나씩 꺼내며, 그 벽이 서로 다른 두 영역을 잇는다면 뚫는다. 이미 연결된 두 영역 사이 벽은 뚫지 않는다(그러면 순환이 생겨 완전연결 미로가 깨진다). 문제는 "두 셀이 이미 같은 영역인가?"를 어떻게 빨리 아느냐다. 그 답이 union-find(분리집합, disjoint set) 다.

union-find는 두 연산만 제공한다.

  • Find(x) — x가 속한 집합의 대표를 찾는다.
  • Union(a, b) — 두 집합을 합친다.

각 원소는 "부모"를 가리키고, 부모를 따라 올라가면 대표(자기 자신이 부모인 원소)가 나온다. Find 는 그 사슬을 타고 올라가되, 올라가며 경로 압축으로 사슬을 짧게 만든다(다음 조회가 빨라진다).

static int uf_find(int* parent, int a)
{
    while (parent[a] != a) {
        parent[a] = parent[parent[a]];   // 경로 압축 — 할아버지를 부모로 (사슬 단축)
        a = parent[a];
    }
    return a;                            // 대표(뿌리) 반환
}

이제 Kruskal은 단순해진다.

void MazeContent::GenKruskal()
{
    ResetWallsAll(true);  OpenAllCells();

    int parent[NUM_CELLS];
    for (int i = 0; i < NUM_CELLS; i++) parent[i] = i;   // 처음엔 각자 자기 집합

    // 셀 사이 모든 벽을 간선으로 수집
    struct Edge { int a, b, wx, wy; };
    std::vector<Edge> edges;
    /* ... 인접 셀 쌍마다 Edge{셀a, 셀b, 사이벽좌표} 추가 ... */

    // Fisher-Yates 셔플 (1부 E 모듈!)
    for (int i = (int)edges.size()-1; i > 0; i--) {
        int j = rand() % (i+1);
        Edge t = edges[i]; edges[i] = edges[j]; edges[j] = t;
    }

    for (size_t i = 0; i < edges.size(); i++) {
        int ra = uf_find(parent, edges[i].a);
        int rb = uf_find(parent, edges[i].b);
        if (ra != rb) {                         // ★ 다른 집합이면 → 벽 뚫고 합치기
            wall[edges[i].wy][edges[i].wx] = false;
            parent[ra] = rb;                    // Union
        }
        // 같은 집합이면 아무것도 안 함 (순환 방지)
    }
}

핵심은 if (ra != rb) 한 줄이다 — 두 셀의 대표가 다르면(아직 안 이어진 별개 영역이면) 벽을 뚫고 두 집합을 합친다. 대표가 같으면(이미 연결됐으면) 건너뛴다. 모든 간선을 처리하면 전체가 한 집합이 되어, 순환 없이 모든 방이 연결된 완전연결 미로가 된다.

이것이 최소 신장 트리(MST)다. "모든 정점을 순환 없이 잇는 최소 간선 집합"이 신장 트리이고, Kruskal은 그걸 만드는 고전 알고리즘이다. 미로 = 격자 그래프의 신장 트리 — 그래서 Kruskal 미로는 편향 없이 균일하다. union-find는 이 MST뿐 아니라 "연결 요소 판정"이 필요한 모든 곳(네트워크 연결성, 이미지 분할, 게임의 지형 그룹)에 쓰이는 근본 자료구조다. 여기서 손으로 만들어 본 것이 두고두고 재산이 된다.

개념 4 · 분할정복 — Recursive Division

Recursive Division은 앞의 넷과 정반대로 간다. 다른 알고리즘은 "다 막힌 데서 통로를 뚫지만", 이건 다 뚫린 빈 방에서 벽을 세운다. 영역을 벽 하나로 나누되 문(통로) 하나를 남기고, 나뉜 두 하위 영역을 재귀적으로 또 나눈다.

void MazeContent::GenDivision(int gx0, int gy0, int gx1, int gy1)
{
    int w = gx1 - gx0, h = gy1 - gy0;
    if (w < 4 && h < 4) return;              // 최소 크기면 종료 (재귀의 바닥)

    bool horizontal = (h > w);               // 긴 쪽을 가로지르는 벽 (정사각형이면 랜덤)

    if (horizontal) {
        int wy = /* 영역 안 랜덤한 짝수 행 */;   // 벽을 세울 행
        int hx = /* 영역 안 랜덤한 홀수 열 */;   // 문을 낼 자리
        for (int x = gx0+1; x <= gx1-1; x++) wall[wy][x] = true;   // 벽 세우기
        wall[wy][hx] = false;                                       // 문 하나 뚫기

        GenDivision(gx0, gy0, gx1, wy);      // ↑ 위 영역 재귀
        GenDivision(gx0, wy, gx1, gy1);      // ↓ 아래 영역 재귀
    } else {
        /* 세로 벽 + 문, 좌/우 영역 재귀 (대칭) */
    }
}

이것이 분할정복(divide and conquer) 이다 — 큰 문제(전체 영역)를 벽 하나로 두 작은 문제(하위 영역)로 나누고, 각각을 같은 방법으로 재귀적으로 푼다. 영역이 충분히 작아지면(재귀의 바닥 조건 w<4 && h<4) 멈춘다. 벽마다 문을 남기니 완전연결이 유지되고, "큰 방 + 좁은 문"이 반복되는 격자형 미로가 된다.

재귀의 두 얼굴. Backtracker(개념 2)는 재귀적 발상을 명시적 스택으로 풀었고(깊은 재귀 회피), Division은 진짜 재귀 함수로 풀었다. 같은 "분할과 되돌아옴"의 발상을, 문제의 성격과 깊이에 따라 스택 또는 재귀로 구현한다. 분할정복은 정렬(퀵/머지), 이진 탐색, 트리 순회 등 CS 전반의 뼈대다 — 미로는 그 눈에 보이는 실습이다.

개념 5 · 경로탐색 4종 — 같은 문제, 네 가지 태도

미로를 다 만들었으면, 이제 푼다. 어느 미로에서든 Tab 으로 알고리즘을 바꾸고 F 로 푸는 과정을 시각화한다. 네 알고리즘의 성격이 뚜렷이 다르다.

알고리즘 자료구조 최단 보장 탐색 모양 배우는 것
BFS 큐 O (무가중) 물결처럼 균등 확장 너비 우선·최단경로
DFS 스택 X 한 방향 깊이 파고들다 백트랙 깊이 우선 (BFS와 대비)
A* 우선순위 O (허용 휴리스틱) 출구 쪽으로 편향 (적게 탐색) 휴리스틱 탐색
벽 따라가기 없음(방향만) X 오른손을 벽에 대고 우회 무메모리 탐색

BFS가 기준이다. 시작점에서 가까운 칸부터 물결처럼 균등하게 퍼지므로, 목표에 처음 도달한 경로가 반드시 최단이다(무가중 격자에서). 그래서 채점용 최단 걸음수도 BFS로 잰다.

// BFS — 큐로 가까운 것부터 확장, prev 로 경로 기억
std::vector<int> q;  size_t head = 0;      // 배열 큐 (head가 앞)
vis[start] = true;   q.push_back(start);
while (head < q.size()) {
    int cur = q[head++];                   // 큐에서 꺼내기 (dequeue)
    if (cur == goal) break;
    int gx = cur % GRID_W, gy = cur / GRID_W;
    for (int d = 0; d < 4; d++) {          // 네 방향 이웃
        if (!GridOpen(gx, gy, d)) continue;    // 벽이면 건너뜀
        int ni = (gy+DY[d])*GRID_W + (gx+DX[d]);
        if (!vis[ni]) { vis[ni] = true; prev[ni] = cur; q.push_back(ni); }  // 큐에 넣기
    }
}

DFS는 큐 대신 스택을 쓴다 — 한 방향으로 끝까지 파고들다 막히면 되돌아온다. 코드는 BFS와 거의 같지만 q[head++](앞에서 꺼냄)가 stk.back()/pop_back()(뒤에서 꺼냄)으로 바뀔 뿐이다. 자료구조 하나(큐↔︎스택)가 알고리즘의 성격을 통째로 바꾼다 — BFS는 넓게, DFS는 깊게. DFS는 최단을 보장하지 않아, BFS와 나란히 보면 그 차이가 극명하다.

A*는 "출구 쪽으로 편향된 BFS" 다. 각 칸의 "지금까지 온 거리 g"에 "출구까지 예상 거리 h(맨해튼 거리)"를 더한 f = g + h 가 작은 칸부터 확장한다. 그래서 BFS처럼 사방으로 퍼지지 않고 출구 방향으로 쏠려 훨씬 적은 칸만 탐색하고도 최단을 찾는다(허용 휴리스틱이라 최단 보장).

int h = abs(gx - goalX) + abs(gy - goalY);   // 맨해튼 휴리스틱 (남은 거리 추정)
int f = g[c] + h;                             // f 작은 것부터 확장 → 목표 편향

벽 따라가기는 기억이 없다 — 오른손을 벽에 대고 계속 따라가면(우수법), 단순연결 미로에서는 언젠가 출구에 닿는다. 큐도 스택도 방문 배열도 필요 없이 현재 방향만 안다. 우아하지만 최단은 아니고, 루프가 있는 미로에선 실패할 수 있다.

같은 미로에서 네 알고리즘을 바꿔 보면 — BFS의 균등한 물결, A*의 목표를 향한 화살, DFS의 헤맴, 벽 따라가기의 우회가 한눈에 비교된다. 하나의 문제에 여러 알고리즘이 있고, 각각 트레이드오프가 다르다 — 이 감각이 알고리즘을 배우는 이유다.

개념 6 · BFS 최단경로 복원과 채점

BFS가 목표에 닿았을 때, 경로는 어디 있을까? prev 배열에 있다. 각 칸이 "누구로부터 왔는지"(prev[ni] = cur)를 기록해 뒀으므로, 목표에서 시작점까지 prev 를 거꾸로 따라가면 경로가 복원된다.

// 목표에서 prev 를 거꾸로 따라 경로 복원 (역순 → 뒤집기)
int c = goalCell;
while (c != -1) { pathOut.push_back(c); c = prev[c]; }   // goal → ... → start (역순)
// pathOut 을 뒤집으면 start → ... → goal
return (int)pathOut.size() - 1;   // 걸음 수 = 칸 수 - 1 (= 최단)

이 prev 되짚기는 모든 경로탐색의 공통 마무리다 — 탐색은 "닿았다"만 알려 주고, "어떻게 왔는지"는 prev 사슬을 역추적해 복원한다. Battle City의 적 탱크가 플레이어를 추적할 때도 이 방식으로 다음 한 칸을 정한다.

채점은 걸음 효율로 한다 — 플레이어의 걸음 steps 를 BFS 최단 optimalSteps 와 비교한다.

float ratio = (float)optimalSteps / (float)steps;   // 1.0 = 완벽한 최단
if (usedSolve)          stars = 1;                    // 자동 풀이를 봤으면 1★
else if (ratio >= 0.9f) stars = 3;                   // 최단의 90%+ → 3★
else if (ratio >= 0.7f) stars = 2;
else                    stars = 1;

최단으로 갈수록 별점이 높다. "네가 걸은 길이 최선에 얼마나 가까운가"를 BFS가 재는 것 — 알고리즘이 채점관이 된다.

코드 워크스루 — 생성 디스패치와 스테이지 흐름

미로 게임의 한 스테이지가 어떻게 짜이는지 따라가자.

실제 파일: Main/Content/MazeContent.cpp — StartStage, GenerateMaze

void MazeContent::StartStage(int stageIndex)
{
    stage = stageIndex;
    genAlgo = (_EGenAlgo)stageIndex;         // ★ 스테이지 = 생성 알고리즘

    seed = (unsigned)rand() ^ ((unsigned)rand() << 15);   // 재현 가능한 시드
    srand(seed);

    // 생성 → 도달성 검증 → 안 되면 재생성 (안전망, 최대 6번)
    for (int attempt = 0; attempt < 6; attempt++) {
        GenerateMaze(genAlgo);
        optimalSteps = ShortestPath(startCell, goalCell, tmp);   // BFS로 검증 + 최단 산출
        if (optimalSteps > 0) break;         // 도달 가능하면 확정
    }
    pgx = sgx; pgy = sgy;  steps = 0;  gameState = _EGameState::PLAYING;
}

void MazeContent::GenerateMaze(_EGenAlgo algo)
{
    switch (algo) {                          // enum으로 알고리즘 디스패치
    case _EGenAlgo::BINARY_TREE: GenBinaryTree(); break;
    case _EGenAlgo::BACKTRACKER: GenBacktracker(); break;
    case _EGenAlgo::PRIM:        GenPrim();        break;
    case _EGenAlgo::KRUSKAL:     GenKruskal();     break;
    case _EGenAlgo::DIVISION:    /* 경계 세팅 후 */ GenDivision(...); break;
    }
}

세 가지가 설계의 핵심이다.

  • enum 으로 알고리즘 디스패치. switch(genAlgo) 하나가 다섯 생성 함수로 갈라진다. 1부 F 모듈의 enum + switch 가, 여기선 "알고리즘을 값으로 고르는" 도구가 된다. 스테이지 = enum 인덱스라, "다음 스테이지"는 StartStage(stage+1) 로 다음 알고리즘을 부른다.
  • BFS가 검증관 겸 채점관. 생성 직후 ShortestPath(BFS)로 시작→출구가 이어지는지 검증하고(완전연결 알고리즘이라 이론상 늘 통과하지만, 인덱싱 실수 방어), 그 결과인 최단 걸음을 채점에 쓴다. 하나의 BFS가 검증·최단·힌트·풀이 시각화에 모두 재사용된다.
  • 시드로 재현. 미로 시드를 저장·표시해, R 키로 새 시드 미로를 뽑거나 같은 시드를 재현할 수 있다(1부 E 모듈 "재현 가능한 랜덤").

OnInGameUpdate 는 상태(GENERATING/PLAYING/SOLVING/CLEAR/PAUSE)에 따라 갈라진다 — PLAYING에선 방향키로 한 칸씩(DAS 반복, Tetris식) 이동하고 벽이면 무시하며, H(힌트)·F(풀이 시각화)·Tab(알고리즘 전환)을 처리한다. 이동은 GridOpen 으로 벽을 검사한다.

void MazeContent::MovePlayer(int dir)
{
    if (!GridOpen(pgx, pgy, dir)) return;   // 벽이면 이동 무시
    pgx += DX[dir];  pgy += DY[dir];  steps++;
    if (pgx == egx && pgy == egy) FinishStage();   // 출구 도달 → 채점
}

알려진 이슈에서 배우기 (as-built)

Maze는 알고리즘 학습이 목적이라, 성능보다 명료함을 택한 부분들이 있다.

이슈 as-built 상태 무엇을 배우나
A 오픈리스트 선형 탐색* 우선순위 큐 대신 매번 최소 f를 선형 탐색 격자가 작아 충분. 진짜 우선순위 큐(힙)는 심화
DFS는 최단 아님 일부러 최단 안 나오게 둠 BFS와 대비해 "왜 BFS가 최단인가"를 가르치는 장치
벽 따라가기 실패 가능 루프 있는 미로(braided)면 못 풀 수 있음 "무메모리 탐색의 한계" — 완전연결 미로에서만 보장
생성 검증 재시도 도달 불가 시 최대 6번 재생성 이론상 불필요(완전연결)하지만 인덱싱 버그 방어망

"DFS가 최단이 아닌 것"과 "벽 따라가기가 실패할 수 있는 것"은 버그가 아니라 교재다. 완벽한 알고리즘만 보여 주면 "왜 여러 알고리즘이 있는가"를 못 배운다. 각 알고리즘의 한계를 함께 보여 주는 것 — 그것이 트레이드오프를 이해하는 유일한 길이다.

Q5. A*가 BFS보다 적은 칸을 탐색하고도 최단을 찾는 비결은? f = g + h 에서 h 는 무엇인가?

해설 펼치기

A1. 방(셀) 사이에 벽이 놓일 자리가 필요하기 때문이다. 셀은 홀수 좌표(항상 통로), 벽은 짝수 좌표(뚫거나 막음)에 둔다. 셀 (cx,cy) 는 격자 (2cx+1, 2cy+1)(둘 다 홀수)에 있고, 두 셀 사이 벽은 그 중간 짝수 좌표다.

A2. 각 셀에서 "북 또는 동"만 뚫는데, 맨 위 행은 북이 없어 항상 동만 뚫리고(위 행이 일자 통로), 맨 오른쪽 열은 동이 없어 항상 북만 뚫린다. 이 경계의 강제된 선택이 미로를 우상단으로 쏠리게 한다 — 알고리즘의 단순한 규칙이 낳는 편향이다.

A3. Find(a) == Find(b) 는 두 셀이 이미 연결돼 있다는 뜻이다. 그런데도 벽을 뚫으면 두 셀 사이에 또 다른 경로가 생겨 순환(cycle) 이 만들어진다. 완전연결 미로(경로가 유일한 트리)가 깨지고, 벽 따라가기 같은 알고리즘이 실패할 수 있다. 그래서 이미 같은 집합이면 건너뛴다.

A4. BFS는 시작점에서 가까운 칸부터 균등하게 확장하므로(큐), 목표에 처음 도달한 순간이 곧 최단이다. 모든 간선 비용이 같은 무가중 격자에서 "가까운 것부터"가 "최단"과 일치한다. DFS는 스택으로 한 방향 깊이 파고들어, 최단이 아닌 먼 길로 먼저 목표에 닿을 수 있다.

A5. A*는 "지금까지 온 거리 g"에 "출구까지 남은 거리 추정 h(맨해튼 거리)"를 더한 f=g+h 가 작은 칸부터 확장한다. h 가 탐색을 출구 방향으로 편향시켜, 엉뚱한 방향은 덜 보고 목표 쪽만 집중적으로 확장한다. h 가 실제보다 크지 않은 "허용 휴리스틱"이면 최단이 보장된다.

실습 미션 — BFS 최단경로 bfs.c ⭐

이 미션은 경로탐색의 심장 BFS를 손으로 만든다. 작은 미로에서 시작점→출구 최단경로를 찾아 prev 로 복원해 그린다 — Maze의 채점, Battle City의 적 AI가 모두 이 코드 위에 선다.

드릴 문법 — 2D 격자 · 큐(배열) · 방문 배열 · 4방향 이웃 · prev 로 최단경로 복원 만드는 것 — 벽/통로로 된 미로에서 S→E 최단경로를 BFS로 찾아, 경로를 * 로 표시해 출력. 클리어 조건 — 최단 걸음 수를 정확히 세고, 경로를 복원해 그린다. 규모 — 약 60줄, 파일 1개. 표준 C만. 알고리즘의 정석을 눈으로 확인한다.

문제 정의

BFS의 핵심은 세 가지다. ① 큐로 가까운 칸부터 확장하고, ② 방문한 칸을 표시하며, ③ 각 칸이 "누구로부터 왔는지"(prev)를 기록한다. 목표에 닿으면 prev 를 거꾸로 따라 경로를 복원한다. 가까운 것부터 확장하니 처음 닿은 경로가 최단이다 — 이 성질을 코드로 확인하는 게 목표다.

목표 / 클리어 조건

  1. 미로를 문자 격자로 둔다(#=벽, .=통로, S=시작, E=출구).
  2. S에서 BFS: 큐에 넣고, 꺼내며 네 방향 이웃 중 통로·미방문 칸을 큐에 넣는다.
  3. 각 칸의 prev(이전 칸 인덱스)를 기록한다.
  4. E에 닿으면 prev 를 거꾸로 따라 경로를 복원, * 로 표시.
  5. 최단 걸음 수를 출력한다.

힌트

힌트 1 — 배열 큐 BFS 
int queue[H*W], head = 0, tail = 0;
int prev[H*W];
int visited[H*W] = {0};

queue[tail++] = startIdx;  visited[startIdx] = 1;  prev[startIdx] = -1;
while (head < tail) {
    int cur = queue[head++];          /* dequeue */
    int r = cur / W, c = cur % W;
    for (int d = 0; d < 4; d++) {
        int nr = r + DR[d], nc = c + DC[d];
        if (막힘 or 방문) continue;
        int ni = nr*W + nc;
        visited[ni] = 1;  prev[ni] = cur;  queue[tail++] = ni;   /* enqueue */
    }
}
힌트 2 — 경로 복원

E에서 prev 를 거꾸로 따라 S까지 가며 * 로 표시하고 걸음을 센다.

int steps = 0;
for (int c = endIdx; c != -1 && c != startIdx; c = prev[c]) {
    maze[c / W][c % W] = '*';   /* 경로 표시 */
    steps++;
}

정답 코드

정답 펼치기 — bfs.c 
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>
#include <string.h>

#define H 7
#define W 11

/* # = 벽, . = 통로, S = 시작, E = 출구 */
char maze[H][W + 1] = {
    "###########",
    "#S..#....E#",
    "#.#.#.###.#",
    "#.#...#...#",
    "#.#####.#.#",
    "#.......#.#",
    "###########",
};

static const int DR[4] = { -1, 0, 1, 0 };   /* 북 동 남 서 */
static const int DC[4] = { 0, 1, 0, -1 };

int main(void)
{
    int queue[H * W], head = 0, tail = 0;
    int prev[H * W], visited[H * W];
    int startIdx = -1, endIdx = -1;

    memset(visited, 0, sizeof(visited));
    for (int r = 0; r < H; r++)
        for (int c = 0; c < W; c++) {
            if (maze[r][c] == 'S') startIdx = r * W + c;
            if (maze[r][c] == 'E') endIdx   = r * W + c;
        }

    /* --- BFS --- */
    queue[tail++] = startIdx;
    visited[startIdx] = 1;
    prev[startIdx] = -1;

    while (head < tail) {
        int cur = queue[head++];               /* 큐에서 꺼내기 (가까운 것부터) */
        if (cur == endIdx) break;
        int r = cur / W, c = cur % W;
        for (int d = 0; d < 4; d++) {
            int nr = r + DR[d], nc = c + DC[d];
            if (nr < 0 || nr >= H || nc < 0 || nc >= W) continue;
            int ni = nr * W + nc;
            if (maze[nr][nc] == '#' || visited[ni]) continue;   /* 벽·방문 제외 */
            visited[ni] = 1;
            prev[ni] = cur;                    /* '누구로부터 왔는가' 기록 */
            queue[tail++] = ni;                /* 큐에 넣기 */
        }
    }

    if (!visited[endIdx]) { printf("경로 없음\n"); return 0; }

    /* --- prev 를 거꾸로 따라 최단경로 복원 --- */
    int steps = 0;
    for (int cur = prev[endIdx]; cur != -1 && cur != startIdx; cur = prev[cur]) {
        maze[cur / W][cur % W] = '*';          /* 경로 표시 */
        steps++;
    }

    printf("최단 걸음: %d\n\n", steps + 1);     /* +1: E로 들어가는 마지막 걸음 */
    for (int r = 0; r < H; r++)
        printf("  %s\n", maze[r]);
    return 0;
}

빌드: cl bfs.c (MSVC) 또는 gcc bfs.c -o bfs

함정/체크포인트

  • 큐는 앞에서 꺼낸다(queue[head++]). 이것이 "가까운 것부터"를 만들어 최단을 보장한다. 만약 뒤에서 꺼내면(stack.pop) DFS가 되어 최단이 깨진다 — 큐↔︎스택 하나가 알고리즘을 바꾼다(개념 5).
  • visited 표시는 큐에 넣을 때. 꺼낼 때 표시하면 같은 칸이 큐에 여러 번 들어가 비효율·중복이 생긴다.
  • prev 가 경로의 전부다. 탐색은 "닿았다"만 알고, 실제 경로는 prev 를 역추적해 복원한다 — 모든 경로탐색의 공통 마무리.

확장 과제

  • 큐를 스택으로 바꿔 DFS로 만들어 보고, 같은 미로에서 경로가 더 길어지는지 비교(개념 5).
  • prev 대신 dist(시작에서의 거리) 배열을 채우면, 모든 칸의 최단 거리를 한 번에 얻는다(BFS의 또 다른 쓰임).
  • union-find로 "두 칸이 연결돼 있나?"를 O(α)에 답하는 union.c 에 도전(개념 3).
