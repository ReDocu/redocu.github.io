## How I Work

- **Documentation comes first.** Whatever I build gets written up as technical or study documentation, and that documentation is deployed so it can be read directly on the site.
- **I build my own tools.** When I notice work repeating itself, I turn it into an automation tool — ClaudeCockpit, the automation tool and the asset manager all started that way.
- **I ship all the way.** Nothing stops at "runs on my machine"; it goes out on Vercel or GitHub Pages until it is actually in use.
