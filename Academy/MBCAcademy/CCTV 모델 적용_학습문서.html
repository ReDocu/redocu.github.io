01

프로젝트 개요

라즈베리파이에 연결한 웹캠 영상을 서버로 받아 실시간으로 객체를 탐지하고, 결과를 웹사이트에서 확인할 수 있게 만들었다. 관리자가 원할 때마다 모델을 바꿔 끼우면 탐지 대상이 동적으로 변경되는 것이 이 프로젝트의 핵심 아이디어. 웹사이트 자체도 로그인·게시판·댓글·좋아요를 갖춘 SNS 형태로 개발해 서버-클라이언트 구조를 함께 학습했다.

구성 요소내용
입력라즈베리파이 + 웹캠 (CCTV 역할, 영상 스트림 전송)
추론라벨링 데이터로 학습시킨 Object Detection 모델 (파일 포맷에 맞게 변환·적용)
서버Flask — 영상 스트리밍 라우트 + SNS 기능(로그인·게시판·댓글·좋아요)
관리모델 정보를 DB에 등록, 선택한 모델로 탐지 대상을 동적 교체
02

데이터 라벨링 경험

최종 프로젝트에 앞서 Object Detection 학습 데이터를 직접 만드는 라벨링 실습을 두 차례 진행했다. 영상 프레임마다 대상에 바운딩 박스를 그리고 클래스를 지정하는 작업이다.

실습대상포인트
라벨링 ① 손흥민 추적하기 경기 영상에서 단일 인물 라벨링 — 빠른 움직임·가림(occlusion) 상황에서 일관된 박스 그리기
라벨링 ② 르브론 제임스 · 스테판 커리 추적하기 다중 클래스 라벨링 — 비슷한 복장의 선수 사이에서 두 대상을 구분하는 기준(유니폼·등번호) 정의
🏷️

체감한 것라벨링 기준이 흔들리면 모델도 흔들린다. 박스를 얼마나 타이트하게 칠지, 가려진 대상을 포함할지 같은 규칙을 팀이 먼저 합의해야 데이터 품질이 유지됐다.

03

시스템 구성

S1영상 입력

라즈베리파이 웹캠이 CCTV처럼 실시간 프레임 캡처

S2모델 추론

서버가 프레임마다 객체 탐지 실행, 바운딩 박스 표시

S3스트리밍

Flask 라우트가 처리된 프레임을 웹 페이지로 연속 전송

S4모델 교체

DB에 등록된 모델 중 선택 → 탐지 대상 동적 변경

04

핵심 학습 개념

Object Detection

  • 분류를 넘어 "무엇이 어디에 있는지"까지 찾는 문제
  • 바운딩 박스 + 클래스 + 신뢰도로 결과 표현
  • 라벨링 품질 → 탐지 품질로 직결되는 구조 이해

모델 파일 포맷 적용

  • 학습 환경과 서비스 환경의 모델 포맷이 다르다는 문제 인식
  • 다양하게 학습시킨 모델을 서비스 포맷에 맞게 변환·적용
  • 모델을 파일 단위로 관리하면 교체가 배포 없이 가능

Flask 웹 개발

  • 라우팅·템플릿·세션으로 SNS 웹사이트 구축
  • 로그인, 게시판, 댓글, 좋아요 기능 구현
  • DB 연동으로 서버-클라이언트 구조의 기본기 학습

실시간 스트리밍

  • 프레임을 연속 응답으로 보내는 MJPEG 스트리밍 방식
  • 추론 속도와 전송 속도의 균형(프레임 스킵) 고민
  • 엣지 장비(라즈베리파이)와 서버의 역할 분담 경험
05

구현 노트

실시간 탐지 스트리밍과 모델 동적 교체의 핵심 아이디어를 요약한 예시 코드.

In [1]Flask 영상 스트리밍 — 탐지 결과를 프레임 단위로 전송
def gen_frames():
    while True:
        ok, frame = camera.read()          # 라즈베리파이 웹캠 프레임
        if not ok: break
        frame = detector.draw_boxes(frame) # 현재 모델로 객체 탐지
        _, buf = cv2.imencode(".jpg", frame)
        yield (b"--frame\r\nContent-Type: image/jpeg\r\n\r\n" + buf.tobytes() + b"\r\n")

@app.route("/video_feed")
def video_feed():
    return Response(gen_frames(), mimetype="multipart/x-mixed-replace; boundary=frame")
In [2]모델 동적 교체 — DB에 등록된 모델을 골라 탐지 대상 변경
@app.route("/models/<int:model_id>/activate", methods=["POST"])
def activate_model(model_id):
    row = db.get_model(model_id)            # 모델 경로·포맷 정보
    detector.load(row.path, fmt=row.format) # 포맷에 맞게 로드
    return {"active": row.name}          # 이후 스트림부터 새 모델로 탐지
06

배운 점 & 회고

  • AI는 모델이 아니라 시스템이다. 라벨링 → 학습 → 포맷 변환 → 서버 적용 → 웹 표시까지, 모델은 전체 파이프라인의 한 조각이었다.
  • 라벨링 품질이 곧 모델 품질. 손흥민·르브론·커리 실습에서 세운 라벨링 규칙의 중요성이 최종 프로젝트에서 그대로 증명됐다.
  • 포맷 문제는 실무의 벽. 학습시킨 모델이 그대로 서비스에 안 들어간다는 것, 변환·적용 방안을 찾는 과정이 실제 개발의 큰 부분이었다.
  • 웹은 AI의 창구. Flask로 SNS 기능까지 만들며, 모델을 사람에게 보여주는 인터페이스의 가치를 배웠다.
💡

한 줄 정리과정 전체(파이썬 → 데이터 분석 → 모델 학습 → 라벨링)의 결과를 하나의 서비스로 묶은 프로젝트. "학습시킨 모델을 실제로 쓰이게 만드는 일"이 무엇인지 처음부터 끝까지 경험했다.
