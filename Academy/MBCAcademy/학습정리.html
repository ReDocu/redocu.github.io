프로그래밍은 값을 이름 붙여 저장하는 것에서 시작한다. 파이썬은 자료형을 선언하지 않아도 값에 따라 자동으로 정해지는 동적 타이핑 언어다.

1-3클래스와 게임 루프

클래스는 데이터(속성)와 동작(메서드)을 하나로 묶는 설계도다. 게임의 플레이어·몬스터·총알은 전부 "위치를 갖고, 움직이고, 그려진다"는 공통점이 있어 클래스로 표현하기 좋다. 그리고 모든 게임은 입력 → 갱신 → 렌더를 초당 수십 번 반복하는 게임 루프로 돌아간다.

    def update(self, keys):             # 메서드: 동작
        if keys[pygame.K_LEFT]:  self.x -= 5
        if keys[pygame.K_RIGHT]: self.x += 5

CH.3

지도 학습 · 딥러닝 — 이미지 분류

학습 목표

  • 지도 학습이 무엇인지, 데이터와 라벨의 관계를 설명할 수 있다
  • CNN이 이미지의 특징을 뽑아내는 원리를 이해한다
  • 학습 곡선을 읽고 과적합을 진단하며, 데이터 증강으로 개선할 수 있다

3-1머신러닝의 세 갈래

방식재료배우는 것예시
지도 학습입력 + 정답(라벨)입력 → 정답의 규칙늑대/허스키 분류, 가격 예측
비지도 학습입력만데이터 속 숨은 구조비슷한 게임끼리 군집화
강화 학습행동 + 보상보상을 최대화하는 전략게임 AI

이 챕터는 지도 학습이다. "이 사진은 늑대(0), 이 사진은 허스키(1)"라는 정답이 붙은 데이터를 주고, 모델이 그 규칙을 스스로 찾게 한다.

3-2신경망과 CNN의 직관

신경망은 입력에 가중치를 곱하고 더한 뒤 활성화 함수를 통과시키는 뉴런의 연결이다. 이미지에는 CNN(합성곱 신경망)을 쓴다 — 합성곱(Conv) 필터가 귀·눈·털 무늬 같은 지역적 특징을 훑어내고, 풀링(Pooling)이 크기를 줄이며 중요한 것만 남긴다. 층이 깊어질수록 "선 → 형태 → 얼굴"처럼 특징이 조합된다.

In [1]CNN 모델 구성 — Conv-Pool 블록을 쌓고 sigmoid로 마무리
model = Sequential([
    Conv2D(32, (3, 3), activation="relu", input_shape=(128, 128, 3)),
    MaxPooling2D(2, 2),          # 특징 요약 (크기 절반)
    Conv2D(64, (3, 3), activation="relu"),
    MaxPooling2D(2, 2),
    Flatten(),                      # 2D 특징맵 → 1D 벡터
    Dense(64, activation="relu"),
    Dense(1, activation="sigmoid"),  # 0~1 확률 (0.5 기준 분류)
])
model.compile(optimizer="adam", loss="binary_crossentropy", metrics=["accuracy"])

3-3학습 곡선 읽기와 과적합

모델의 상태는 학습/검증 곡선을 그려봐야 보인다. 학습 정확도만 오르고 검증 정확도가 멈추면 과적합 — 모델이 문제를 이해한 게 아니라 답을 외운 상태다. 처방은 데이터를 늘리거나(증강), 모델을 줄이거나, 일찍 멈추는 것.

In [2]데이터 증강 + 학습 곡선 — 과적합 진단의 기본 도구
train_gen = ImageDataGenerator(rescale=1/255, zoom_range=0.2,
                               horizontal_flip=True)   # 증강: 공짜 데이터

history = model.fit(train, validation_data=val, epochs=20)

데이터 편향 주의늑대 사진에는 눈(雪) 배경이 많다. 데이터가 편향되면 모델은 동물이 아니라 배경을 보고 분류한다. 정확도가 높아도 엉뚱한 것을 배웠을 수 있다 — 데이터를 의심하는 습관이 딥러닝의 절반이다.

학습 목표

  • EDA로 데이터를 탐색하고 가설을 세울 수 있다
  • 이상치 제거·인코딩·분리·정규화를 올바른 순서로 수행할 수 있다
  • 회귀 모델을 지표(MAE·RMSE·R²)로 평가하고 하이퍼파라미터를 튜닝할 수 있다

4-1EDA — 모델링 전에 데이터와 대화하기

EDA(탐색적 데이터 분석)는 히스토그램·산점도·상관계수로 데이터의 분포와 관계를 먼저 살피는 단계다. "어떤 특징이 가격에 영향을 줄까?"라는 가설이 여기서 나오고, 이상치 기준·특징 선택의 근거가 된다. CH.2의 전처리 기술이 그대로 재사용된다.

4-2전처리 4종 세트 — 순서가 중요하다

순서작업이유
①이상치 제거 (IQR 기준)극단값이 모델을 왜곡하지 않도록
②원-핫 인코딩모델은 숫자만 이해한다 — 장르 같은 범주형을 0/1 컬럼으로
③train / test 분리공정한 평가를 위해 시험 문제를 미리 감춘다
④정규화 (분리 후!)분리 전에 하면 테스트 정보가 학습에 새어든다 = 데이터 누수
In [1]전처리 파이프라인 — ①→②→③→④ 순서 그대로
q1, q3 = df["price"].quantile([0.25, 0.75]); iqr = q3 - q1
df = df[(df["price"] >= q1 - 1.5*iqr) & (df["price"] <= q3 + 1.5*iqr)]  # ①
df = pd.get_dummies(df, columns=["genre"])                             # ②

4-3회귀 모델과 평가 지표

연속된 수치(가격)를 예측하는 문제는 회귀다. "좋은 모델"은 감이 아니라 지표로 정의한다.

지표의미읽는 법
MAE평균 절대 오차"평균적으로 얼마나 빗나가는가" — 직관적
RMSE평균 제곱근 오차큰 오차에 더 민감 — 치명적 오차를 잡을 때
R²결정 계수1에 가까울수록 데이터를 잘 설명 (0.7이면 70%)

4-4하이퍼파라미터 튜닝

트리 깊이·트리 개수처럼 모델이 스스로 배우지 못하는 값은 사람이 정한다. GridSearch + 교차검증으로 조합을 체계적으로 탐색한다 — 단, 튜닝의 개선 폭은 전처리 품질을 넘지 못한다. 데이터가 먼저, 튜닝은 마지막이다.

In [2]학습 → 평가 → GridSearch 튜닝
model = RandomForestRegressor().fit(X_train, y_train)
pred = model.predict(X_test)
print(mean_absolute_error(y_test, pred), r2_score(y_test, pred))

grid = GridSearchCV(RandomForestRegressor(),
                    {"n_estimators": [100, 300], "max_depth": [8, 16, None]},
                    cv=5)
grid.fit(X_train, y_train)
print(grid.best_params_)         # 최적 조합 확인

CH.5

Object Detection — 데이터 라벨링

학습 목표

  • 분류와 객체 탐지의 차이를 설명할 수 있다
  • 바운딩 박스·IoU 등 탐지의 기본 개념을 이해한다
  • 일관된 규칙으로 학습용 데이터를 직접 라벨링할 수 있다

5-1분류에서 탐지로

CH.3의 분류는 "이 사진이 무엇인가"에 답한다. Object Detection은 "무엇이, 어디에, 몇 개나 있는가"까지 답한다. 결과는 바운딩 박스(위치) + 클래스(이름) + 신뢰도(확률)의 묶음이다.

개념설명
바운딩 박스대상을 감싸는 사각형 — 좌표 (x, y, w, h)로 표현
IoU예측 박스와 정답 박스가 겹치는 비율 — 탐지가 "맞았다"의 기준
신뢰도모델이 "이 박스 안에 대상이 있다"고 확신하는 정도
클래스박스 안 대상의 이름 — 라벨링 때 정의한 목록에서 나온다

5-2라벨링 실습 — 데이터를 직접 만들다

모델을 "쓰는" 입장에서 데이터를 "만드는" 입장으로 바꿔보는 실습. 영상 프레임마다 대상에 박스를 그리고 클래스를 지정했다.

실습대상난이도 포인트
①손흥민 추적하기단일 인물 — 빠른 움직임, 다른 선수와의 겹침(가림) 처리
②르브론 제임스 · 스테판 커리다중 클래스 — 비슷한 대상을 구분할 기준(유니폼·등번호) 정의

라벨링 전에 팀이 반드시 합의해야 하는 규칙:

  • 박스는 얼마나 타이트하게? 몸 전체인지, 여백 없이 딱 맞게인지 — 기준이 섞이면 모델이 흔들린다.
  • 가려진 대상은 포함하나? 절반 이상 가림, 화면 밖 잘림 등의 처리 규칙을 먼저 정한다.
  • 클래스 구분 기준은? 사람 눈에도 헷갈리는 대상이라면, 구분 근거(등번호 등)를 명문화한다.
🏷️

핵심 교훈데이터 품질 = 모델 품질. 라벨링 기준이 흔들리면 아무리 좋은 모델도 흔들린다. AI 프로젝트에서 라벨링은 잡일이 아니라 성능을 결정하는 설계 작업이다.

CH.6

모델 서비스화 — Flask와 실시간 CCTV

학습 목표

  • Flask로 라우팅·템플릿 기반 웹 서비스를 만들 수 있다
  • 학습시킨 모델을 서비스 포맷에 맞게 적용할 수 있다
  • 실시간 영상 스트리밍에 탐지 모델을 연결할 수 있다

6-1Flask 기초 — 웹은 요청과 응답이다

웹 서버는 결국 "주소(URL)로 요청이 오면 응답을 돌려주는 프로그램"이다. Flask에서는 함수에 @app.route를 붙여 주소와 함수를 연결한다. 이 구조 위에 로그인·게시판·댓글·좋아요를 얹으면 SNS가 된다.

In [1]라우팅 기본 — 주소와 함수를 연결
from flask import Flask, render_template, request

app = Flask(__name__)

@app.route("/")
def index():
    return render_template("index.html")   # 템플릿 렌더링

@app.route("/login", methods=["POST"])
def login():
    user = request.form["username"]         # 폼 데이터 수신
    ...

6-2모델 포맷 — 학습 환경과 서비스 환경은 다르다

노트북에서 학습한 모델 파일이 서버에 그대로 들어가지 않는 경우가 많다. 모델을 서비스 포맷에 맞게 변환·적용하는 일은 실무의 큰 부분이다. 모델을 파일 단위로 관리하고 경로·포맷을 DB에 등록해두면, 배포 없이 모델을 갈아 끼우는 구조를 만들 수 있다.

6-3실시간 스트리밍 — 탐지 결과를 웹으로

라즈베리파이 웹캠의 프레임을 서버가 받아 모델로 탐지하고, 처리된 프레임을 MJPEG 방식(이미지를 연속으로 밀어내는 응답)으로 브라우저에 흘려보낸다.

In [2]탐지 + 스트리밍 + 모델 동적 교체 — 최종 프로젝트의 핵심 구조
def gen_frames():
    while True:
        ok, frame = camera.read()
        frame = detector.draw_boxes(frame)      # 현재 모델로 탐지
        _, buf = cv2.imencode(".jpg", frame)
        yield (b"--frame\r\n" b"Content-Type: image/jpeg\r\n\r\n" + buf.tobytes())

@app.route("/video_feed")
def video_feed():
    return Response(gen_frames(), mimetype="multipart/x-mixed-replace; boundary=frame")

@app.route("/models/<int:mid>/activate", methods=["POST"])
def activate(mid):
    row = db.get_model(mid)
    detector.load(row.path, fmt=row.format)     # 탐지 대상 동적 교체
