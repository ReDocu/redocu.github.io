# Kakuro — Game Design Document

> 웹 브라우저용 가쿠로(Kakuro) 퍼즐 게임
> 참고 자료: 나무위키 「가쿠로」 (2026-07-17 판) · 구현 전제: `web_page/Sudoku`와 동일한 바닐라 HTML/CSS/JS 구조

---

## 1. 개요

| 항목 | 내용 |
|---|---|
| 게임명 | Kakuro (가쿠로 / カックロ) |
| 장르 | 논리 퍼즐 (숫자 크로스워드) |
| 플랫폼 | 웹 브라우저 (데스크톱 + 모바일 반응형) |
| 기술 | HTML / CSS / Vanilla JS — 외부 라이브러리 없음 |
| 플레이 인원 | 1인 |
| 세션 길이 | 판당 5~30분 (크기·난이도에 따라) |

가쿠로는 스도쿠와 기본은 같지만 **덧셈 분해(인수분해식 접근)를 이용한 수학 버전 크로스워드**로 불리는 퍼즐이다. 1966년 델 매거진의 제이콥 펑크(Jacob E. Funk)가 'cross sum'이라는 이름으로 처음 제안했고, 일본에서 '加算クロス(kasan kurosu)'를 줄인 **kakuro**라는 이름으로 대중화되었다.

**디자인 목표**

- 스도쿠 경험자가 규칙을 1분 안에 이해하고 바로 플레이할 수 있을 것
- 종이 퍼즐에서 하던 보조 작업(조합표 찾기, 연필 메모)을 UI가 대신해 줄 것
- 퍼즐은 항상 **유일해**를 보장할 것 (제너레이터 + 솔버 검증)

## 2. 게임 규칙 (원전 기준)

보드는 **흰 칸**(입력), **검은 칸**(차단), **힌트 칸**(대각선으로 분할, 합 숫자 표기)으로 구성된다. 모든 흰 칸에 적절한 값을 채워 넣는 것이 목표다.

1. 모든 흰 칸에는 **1부터 9까지의 숫자**만 써넣을 수 있다.
2. 힌트 칸 **대각선 위의 숫자**는 그 칸의 오른쪽 방향으로, 다음 검은/힌트 칸 사이에 있는 모든 흰 칸 숫자의 **합**과 일치해야 한다. (가로 런)
3. 힌트 칸 **대각선 아래의 숫자**는 그 칸의 아래 방향으로, 다음 검은/힌트 칸 사이에 있는 모든 흰 칸 숫자의 **합**과 일치해야 한다. (세로 런)
4. 한 줄(런)을 이루는 흰 칸에는 **같은 수를 두 번 넣을 수 없다.** 힌트가 4라면 1+3 또는 3+1은 가능하지만 2+2는 불가능하다.

원전 기준 표준 크기는 16×16이지만 얼마든지 커지거나 작아질 수 있다 (→ §6 난이도).

**파생 성질** (풀이·생성 로직의 기반)

- 한 런의 합은 최소 3(2칸: 1+2), 최대 45(9칸: 1~9)이다.
- 런 길이가 n이면 합의 범위는 `n(n+1)/2` ~ `n(19-n)/2`.

## 3. 핵심 플레이 루프

```
퍼즐 선택(크기·난이도)
   ↓
칸 선택 → 후보 확인(조합 도우미) → 숫자 확정 or 메모
   ↓                                    ↑
런 완성 시 즉시 검증 피드백 ────────────┘
   ↓
전체 완성 → 클리어 연출 + 기록(시간·힌트 사용 수)
```

플레이어의 실제 사고 과정은 나무위키 「기술」 섹션의 정석 풀이를 따른다:

1. **유일 조합부터**: 두 칸짜리 3, 4, 16, 17처럼 조합이 하나뿐인 런을 먼저 공략한다.
2. **교차 배제**: 세로 두 칸짜리 16(7,9)을 보고, 그 칸을 공유하는 가로 런에서 7과 9가 들어가지 않는 조합을 우선적으로 배제한다.
3. **후보 소거**: 쉬운 것부터 확실히 풀면서 각 흰 칸에 들어갈 수 있는 숫자를 하나씩 줄여나가고, 불가능한 요소를 모두 없애 남은 것이 정답이다.

게임 UI는 이 세 단계를 각각 **조합 도우미 / 자동 후보 표시 / 메모** 기능으로 지원한다 (→ §5).

## 4. 게임 구성 요소

### 4.1 보드

| 요소 | 설명 |
|---|---|
| 흰 칸 (cell) | 플레이어가 1~9를 입력. 메모(후보 숫자) 겹쳐 표시 가능 |
| 검은 칸 (block) | 입력 불가. 런의 경계 |
| 힌트 칸 (clue) | 대각선 분할. 우상단 = 가로 합, 좌하단 = 세로 합. 한쪽만 있을 수도 있음 |

### 4.2 런(run)

- 가로/세로로 연속된 흰 칸의 묶음. 퍼즐의 최소 검증 단위.
- 내부 데이터로 `{방향, 시작좌표, 길이, 목표합, 소속 셀 목록}`을 가진다.
- 모든 흰 칸은 정확히 가로 런 1개 + 세로 런 1개에 속한다.

### 4.3 족보 (조합표)

나무위키 「족보」 섹션의 합·칸수별 가능 조합 전체를 데이터화한다. 특히 **조합이 단 하나뿐인 매직 넘버**는 힌트 시스템의 핵심:

| 칸수 | 합 → 유일 조합 |
|---|---|
| 2칸 | 3=(1,2) · 4=(1,3) · 16=(7,9) · 17=(8,9) |
| 3칸 | 6=(1,2,3) · 7=(1,2,4) · 23=(6,8,9) · 24=(7,8,9) |
| 4칸 | 10=(1,2,3,4) · 11=(1,2,3,5) · 29=(5,7,8,9) · 30=(6,7,8,9) |
| 5칸 | 15=(1~5) · 16=(1,2,3,4,6) · 34=(4,6,7,8,9) · 35=(5,6,7,8,9) |
| 8칸 | 36~44 = 각각 유일 (45에서 빠지는 숫자 1개로 결정) |
| 9칸 | 45=(1~9) |

조합표는 하드코딩하지 않고 시작 시 한 번 생성해 캐싱한다 (`combinations(sum, length)` — 전체 경우가 작아 비용 무시 가능).

응용 팁도 힌트 로직에 반영한다: 예를 들어 "16, 2칸"과 "23, 3칸"이 교차하면 **공유 가능한 숫자는 9뿐**이므로 교차 칸이 9로 확정된다 (두 족보의 교집합 계산으로 일반화).

## 5. 기능 명세

### 5.1 입력

| 기능 | 데스크톱 | 모바일 |
|---|---|---|
| 칸 선택 | 클릭 / 방향키 이동 | 탭 |
| 숫자 입력 | 키보드 1~9 / 화면 숫자패드 | 화면 숫자패드 |
| 메모 토글 | `N` 키 또는 패드의 연필 버튼 | 연필 버튼 |
| 지우기 | `Del`/`BS` / 패드 X | 패드 X |
| 실행 취소 | `Ctrl+Z` (다중 단계) | 버튼 |

### 5.2 보조 기능 (종이 퍼즐의 고통 제거)

- **조합 도우미**: 칸을 선택하면 소속된 가로/세로 런의 `합·칸수·남은 조합 목록`을 사이드 패널에 표시. 이미 입력된 숫자와 모순되는 조합은 취소선 처리.
- **자동 후보 모드(옵션)**: 각 빈 칸에 가능한 후보를 자동 계산해 연하게 표시. 기본 OFF (수동 메모가 기본 재미).
- **오류 표시(옵션)**: ① 런 합 초과/중복 즉시 경고 ② 정답과 다른 입력 즉시 표시 — 난이도 설정과 연동, 기본은 ①만.
- **힌트 버튼**: 1회당 논리적으로 확정 가능한 칸 하나를 근거와 함께 공개 ("이 세로 런 16(2칸)은 (7,9)뿐이고 가로 런에 9가 이미 있으므로 7"). 사용 횟수는 기록에 남긴다.

### 5.3 퍼즐 생성 · 검증

1. 크기에 맞는 검은 칸 패턴 생성 (180° 회전 대칭, 고아 칸·길이 1 런 금지, 생성기는 런을 7칸으로 제한 — 긴 런은 조합 폭발로 유일해가 잘 나오지 않음)
2. 흰 칸에 규칙을 만족하는 정답 숫자 채우기 (백트래킹 + **극단 숫자(1·2·8·9) 편향** — 합이 극값에 가까울수록 족보 조합이 급감해 유일해 확률이 오름)
3. 정답에서 합 힌트 추출 → 숫자는 전부 지움
4. **전파 솔버로 검증** — 전파만으로 전 칸이 확정되면 그 자체로 유일해가 증명되며(결정적 절차), 사람이 찍기 없이 풀 수 있는 판임도 동시에 보장된다
5. 확정 실패 시 **국소 수리**: 미확정 칸(모호 영역)만 다시 채워 그 지점의 힌트 합을 바꾸고 재검증. a-b/b-a 사각형(스도쿠의 deadly pattern에 해당)은 값 교환만으로 해가 2개가 되므로 사전 필터로 먼저 제거한다

솔버는 사람의 풀이를 모사한 단계형(유일 조합 → 교차 교집합 → 후보 소거 → 최후 백트래킹)으로 만들어 유일해 검증·힌트 문구 생성·자동 후보 계산에 재사용한다. 전역 해 개수 탐색(백트래킹)은 16×16에서 비용이 폭발하므로 생성 루프에는 쓰지 않고 검증 테스트용으로만 남긴다.

### 5.4 저장

- `localStorage`에 진행 중인 판(보드·메모·경과 시간)과 클리어 기록 저장. 새로고침 후 이어하기 지원.

## 6. 난이도 설계

원전의 변형 섹션(4×4부터 30×30까지)과 kakuroconquest.com의 5단계 난이도(초급·중급·고급·어려운·전문가)를 참고하되, 웹 플레이에 맞게 **크기 × 논리 깊이** 매트릭스로 재구성한다.

| 프리셋 | 크기 | 요구 논리 | 비고 |
|---|---|---|---|
| 입문 | 6×6 | 유일 조합만으로 완주 가능 | 튜토리얼 겸용 |
| 초급 | 8×8 | 유일 조합 + 단순 교차 배제 | |
| 중급 | 10×10 | 교차 교집합 다단 적용 | 기본 선택값 |
| 고급 | 12×12 | 깊은 후보 소거 | |
| 전문가 | 16×16 | 전 기법 총동원 | 원전 표준 크기 |

- 같은 크기라도 솔버 난이도 점수로 세부 조정한다 (매직 넘버 런의 비율이 낮을수록 어려움).
- 최초 플레이 시 입문 판 + 규칙 오버레이(§2의 4개 규칙 그림 설명)를 노출한다.

## 7. UI / 화면 설계

```
┌────────────────────────────────────────────┐
│  Kakuro      [난이도▾] [새 게임] ⏱ 12:34   │
├───────────────────────────┬────────────────┤
│                           │  선택 런 정보    │
│                           │  가로 23 (3칸)   │
│        보드 그리드          │  · 6 8 9        │
│   (힌트 칸 대각선 렌더링)     │  · ~5 9~ (배제)  │
│                           ├────────────────┤
│                           │  [1~9 숫자패드]  │
│                           │  [✏메모][⌫][↩]  │
│                           │  [💡힌트]        │
└───────────────────────────┴────────────────┘
```

- **힌트 칸 렌더링**: CSS `linear-gradient` 대각선 + 두 숫자 절대 배치 (이미지 없이 구현).
- 선택 칸 하이라이트 + 같은 런 전체를 연한 색으로 강조 (스도쿠의 행/열 강조와 동일한 문법).
- 모바일(≤520px): 사이드 패널이 보드 아래로 내려오고, 조합 도우미는 접이식.
- 클리어 시: 보드 완성 애니메이션 + 시간/힌트 사용/난이도 요약 카드.

## 8. 기술 구조

`web_page/Sudoku`와 동일한 파일 분할 컨벤션을 따른다.

```
web_page/Kakuro/
├── GDD.md               ← 이 문서
├── index.html
├── css/
│   └── style.css
└── js/
    ├── combinations.js   # 족보 생성·캐싱, 교집합 유틸
    ├── kakuroGenerator.js# 패턴 생성 → 정답 채우기 → 힌트 추출
    ├── kakuroSolver.js   # 단계형 솔버 (유일해 검증·난이도 측정·힌트 근거)
    ├── ui.js             # 보드 렌더링, 입력 처리, 패널
    └── main.js           # 게임 상태, 저장/로드, 진입점
```

**핵심 데이터 모델**

```js
// 셀: { type: 'white'|'block'|'clue', value, notes:Set, acrossClue, downClue }
// 런: { dir:'across'|'down', cells:[...], sum, candidates:[[...조합]] }
// 게임: { grid, runs, difficulty, elapsed, history:[] }
```
