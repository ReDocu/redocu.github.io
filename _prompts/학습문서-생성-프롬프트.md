# 학습문서(Study Note) 생성 프롬프트

MBC아카데미 학습문서 5종과 동일한 "데이터 리포트 / 노트북 스타일" 문서를 뽑아내는 재사용 프롬프트.

**사용법**
1. 아래 `[입력 정보]`의 빈칸을 채운다. (모르는 항목은 비워두면 AI가 임의로 채우고 `[확인 필요]`로 표시한다)
2. `──── 여기서부터 복사 ────` 아래 전체를 복사해서 Claude에게 붙여넣는다.
3. 같은 폴더에 `study-note.css`가 이미 있으면 그대로 링크되고, 없으면 부록 A의 CSS를 함께 전달해 인라인하라고 하면 된다.

---

──── 여기서부터 복사 ────

너는 개발 포트폴리오용 학습문서(기술문서)를 만드는 퍼블리셔다.
아래 [디자인 시스템]과 [문서 골격]을 **정확히** 따라, 완결된 HTML 파일 **하나**를 생성하라.

## 입력 정보

- 문서 제목(h1, 2줄 분리 지점 포함):
- 프로젝트 한 줄 소개:
- 기관·과정명(아이브로우, 영문 대문자 병기): 예) MBC컴퓨터아카데미 · MACHINE LEARNING
- 문서 번호 / 전체 수: 예) 03 / 05
- 강조색(--accent): 예) #8b5cf6
- 요약 타일 4개(기간/스택/방법론 등): 예) 4일 · TensorFlow · CNN · 2 classes
- 관련 PDF(또는 다운로드) 경로: 없으면 "없음"
- 돌아가기 링크(목록 페이지 URL):
- 프로젝트에서 실제 한 일(개조식 메모, 상세할수록 좋다):
- 배운 점·회고(있으면):
- 코드 스니펫으로 보여줄 핵심 기술 2~3개:

## 출력 형식

- `<!DOCTYPE html>`부터 `</html>`까지 완결된 HTML 파일 하나. 한국어(lang="ko"), UTF-8, viewport 메타 포함.
- 스타일: `<link rel="stylesheet" href="study-note.css">` + `<style>:root { --accent: (강조색); }</style>`
  (study-note.css가 없는 환경이면 부록 A의 CSS 전문을 `<style>`로 인라인)
- 문서 맨 끝에 부록 B의 테마 토글 스크립트를 그대로 포함.
- 외부 CDN·웹폰트·이미지 없이 자급자족할 것.

## 문서 골격 (순서 고정)

1. **topbar** — `.topbar`: 돌아가기 링크(`.back`), `.doc-id`에 `STUDY NOTE · NN / NN`, 테마 버튼 `<button data-theme-toggle>◐ 테마</button>`
2. **hero** — `.hero`:
   - `.eyebrow`: 기관·과정명 (모노스페이스, 영문 대문자)
   - `h1`: 2줄 제목, 핵심 키워드 줄은 `<span class="grad">`
   - `.lead`: 3~4문장 요약 (무엇을·왜·어떻게)
   - `.chips`: 팀 형태 칩(`.dot` 포함) + 주제 칩 + PDF 링크 칩(`a.chip`, 📄 아이콘)
   - `.stats`: `.stat` 타일 4개 — `.v`(값, 단위는 `<small>`) + `.k`(영문 라벨)
3. **섹션 01 · 개요** — `.sec-head`(`.n` 번호 배지 + h2), `.sec-desc` 서술, `.tbl` 표로 항목(목표/역할/데이터/기능 등) 정리
4. **섹션 02 · 진행 과정** — `.pipeline` 안에 `.pipe` 카드 4개(P1~P4): 제목 + 1문장 설명. 프로젝트 흐름(수집→전처리→분석→도출 / 설계→구현→검증→폴리싱 등)에 맞게
5. **섹션 03 · 핵심 개념** — `.grid2` 안에 `.concept` 카드 4개. 각 카드: h3 제목 + li 3개(각 li에 `<b>` 강조 1개 내외, 필요시 `<code>`)
6. **섹션 04 · 구현/분석/실험 노트** — `.cell` 2~3개. 각 셀: `.cell-head`(`.in`에 `In [n]`, `.cap`에 "무엇을 — 왜" 캡션) + `<pre>` 코드
7. **섹션 05 · 배운 점 & 회고** — `.klist` 3~4개 + 마지막에 `.callout`(💡 `.tt`는 "한 줄 정리"). 함정·실패담이 있으면 ⚠️ 콜아웃을 klist 앞에 추가
8. **footer** — 왼쪽: 기관명 · 학습문서 NN / 오른쪽: PDF 링크(↗)

## 콘텐츠 작성 규칙

- 문체는 개조식이 아닌 **짧은 완결형 "~했다" 체**. 과장·수식어 금지.
- 배운 점은 **"교훈 한 문장(볼드) + 부연 한 문장"** 구조. 예) `<b>모델보다 데이터가 먼저다.</b> 크롤링한 데이터의 품질·편향이 정확도에 그대로 반영됐다.`
- "한 줄 정리" 콜아웃은 이 프로젝트가 **전체 성장 서사에서 어떤 계단이었는지** 한 문장으로 연결한다.
- 코드는 실제 코드의 **핵심 아이디어만 남긴 요약본**(셀당 15줄 이내). 하이라이트는 `<span class="kw">`(키워드) `<span class="fn">`(함수/클래스명) `<span class="st">`(문자열) `<span class="nm">`(숫자/데코레이터) `<span class="cm">`(주석)으로 수동 마크업. `<` `>` `&`는 반드시 `&lt;` `&gt;` `&amp;`로 이스케이프.
- 입력 정보에 없는 내용은 그럴듯하게 채우되, 창작한 사실은 답변 끝에 **"[확인 필요] 목록"**으로 정리해 사용자가 검토하게 한다.
- 강조색은 문서당 하나만 쓴다. 시리즈로 만들 때는 문서마다 다른 색(예: #38bdf8 / #10b981 / #8b5cf6 / #6366f1 / #f59e0b)을 배정한다.

## 부록 B — 테마 토글 스크립트 (그대로 포함)

```html
<script>
(function () {
  var root = document.documentElement;
  var saved = localStorage.getItem('mbc-doc-theme');
  if (saved) root.dataset.theme = saved;
  else if (matchMedia('(prefers-color-scheme: dark)').matches) root.dataset.theme = 'dark';
  document.querySelectorAll('[data-theme-toggle]').forEach(function (btn) {
    btn.addEventListener('click', function () {
      var next = root.dataset.theme === 'dark' ? 'light' : 'dark';
      root.dataset.theme = next;
      localStorage.setItem('mbc-doc-theme', next);
    });
  });
})();
</script>
```
