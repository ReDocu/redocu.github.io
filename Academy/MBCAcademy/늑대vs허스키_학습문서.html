01

프로젝트 개요

늑대와 허스키는 생김새가 비슷해 이미지 분류의 난이도와 함정을 동시에 보여주는 주제다. 데이터 수집(크롤링)부터 모델 구성, 학습 그래프 검증, 데이터 증강까지 지도 학습의 전체 사이클을 처음부터 끝까지 직접 돌려보는 것이 목적이었다.

항목내용
데이터크롤링으로 수집한 늑대·허스키 이미지 (라벨 = 폴더 구분)
모델Conv2D + MaxPooling을 쌓은 CNN, 마지막은 sigmoid 이진 출력
검증학습/검증 정확도·손실 그래프로 과적합 여부 판단
개선이미지 크기 변형·자르기 등 데이터 증강으로 재학습
02

진행 과정

P1데이터 수집

이미지 크롤링으로 클래스별 데이터셋 구축, 불량 이미지 수작업 제거

P2전처리 · 증강

크기 통일·정규화, 크기 변형·자르기로 데이터 증폭

P3모델 구성

Conv-Pool 블록을 쌓고 sigmoid 출력의 CNN 설계

P4학습 · 검증

epoch별 정확도·손실 그래프를 그려 과적합 확인 및 개선

03

핵심 학습 개념

지도 학습 · 이진 분류

  • 입력(이미지) + 정답(라벨) 쌍으로 모델을 학습
  • 출력층 sigmoid가 0~1 확률을 내고, 0.5를 기준으로 분류
  • 손실 함수는 binary crossentropy 사용

CNN 구조

  • 합성곱(Conv)이 이미지의 지역적 특징(귀·눈·털 무늬)을 추출
  • 풀링(Pooling)으로 크기를 줄이며 중요한 특징만 남김
  • 층을 쌓을수록 단순한 선 → 복잡한 형태 순으로 특징이 조합됨

학습 곡선 검증

  • 학습 정확도만 오르고 검증 정확도가 정체되면 과적합
  • 그래프를 그려봐야 모델의 상태가 보인다는 것을 체득
  • epoch 수·모델 크기를 조정하며 곡선 변화 관찰

데이터 증강

  • 크기 변형·자르기·뒤집기로 같은 이미지에서 여러 학습 샘플 생성
  • 적은 데이터로도 일반화 성능을 끌어올리는 방법
  • 증강 전후의 검증 정확도 차이를 직접 비교
04

실험 노트

모델 구성과 검증 흐름을 요약한 예시 코드.

In [1]데이터 증강 — 크기 변형·자르기·뒤집기로 데이터 증폭
from tensorflow.keras.preprocessing.image import ImageDataGenerator

train_gen = ImageDataGenerator(
    rescale=1/255,          # 정규화
    zoom_range=0.2,          # 확대(자르기 효과)
    width_shift_range=0.1,
    horizontal_flip=True,
).flow_from_directory("data/train", target_size=(128, 128), class_mode="binary")
In [2]CNN 모델 — Conv-Pool 블록 + sigmoid 이진 출력
model = Sequential([
    Conv2D(32, (3, 3), activation="relu", input_shape=(128, 128, 3)),
    MaxPooling2D(2, 2),
    Conv2D(64, (3, 3), activation="relu"),
    MaxPooling2D(2, 2),
    Flatten(),
    Dense(64, activation="relu"),
    Dense(1, activation="sigmoid"),   # 늑대(0) vs 허스키(1)
])
model.compile(optimizer="adam", loss="binary_crossentropy", metrics=["accuracy"])
In [3]학습 곡선 — 그래프로 과적합 여부 확인
history = model.fit(train_gen, validation_data=val_gen, epochs=20)

plt.plot(history.history["accuracy"], label="train")
plt.plot(history.history["val_accuracy"], label="val")
plt.legend()                    # 두 곡선이 벌어지면 과적합 신호
05

배운 점 & 회고

⚠️

함정늑대 사진에는 눈(雪) 배경이 많다. 모델이 동물이 아니라 배경을 보고 분류할 수 있다는 것 — 데이터가 편향되면 모델은 엉뚱한 것을 배운다는 사실을 이 주제로 직접 확인했다.

  • 모델보다 데이터가 먼저다. 크롤링한 데이터의 품질·편향이 정확도에 그대로 반영됐다.
  • 그래프 없이는 판단할 수 없다. 학습/검증 곡선을 그려봐야 과적합인지, 더 학습해야 하는지 알 수 있었다.
  • 증강은 공짜 데이터다. 같은 이미지를 변형하는 것만으로 검증 정확도가 눈에 띄게 올랐다.
  • CNN의 직관 이해. 합성곱이 특징을 뽑고 풀링이 요약한다는 흐름을 코드와 결과로 연결해 이해했다.
💡

한 줄 정리지도 학습의 사이클(데이터 → 모델 → 학습 → 검증 → 개선)을 처음 완주했고, "AI의 성능은 데이터에서 시작된다"는 이후 라벨링 과정의 문제의식을 여기서 얻었다.
