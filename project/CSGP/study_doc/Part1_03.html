C 모듈 · 7–8장 — 배열과 문자열: 회사에 사람과 이름을 채우다

모듈 C · 7–8장 · 게임 버전 v0.3 → v0.4 · 예상 학습 시간 3–5시간 처음 배우는 기술 — 배열 선언·인덱스 배열 초기화 배열 순회 2차원 배열 고정 크기의 한계 char 문자열 널 종료('\0') string.h (strcpy·strlen·strcmp) 문자열 배열 fgets 입력 입력 버퍼 비우기

지금까지 우리 회사는 사장 혼자였고 이름도 없었다. 이 모듈에서 배열로 직원 여럿을 한 목록에 담고, 문자열로 회사와 게임에 이름을 준다. 그런데 아직 구조체(9장)가 없어서, 직원 한 명의 정보를 여러 배열에 나눠 담아야 한다 — 이 방식은 곧 손이 아파 온다. 그 아픔이 바로 다음 D 모듈에서 구조체를 반갑게 만드는 준비운동이다.

이 모듈의 학습 목표

이 모듈을 마치면 다음을 할 수 있다.

  1. 같은 자료형 여럿을 배열에 담고, 인덱스로 접근하며, for 로 순회할 수 있다.
  2. 2차원 배열의 구조를 이해하고, 월별 매출 기록·문자열 목록에 쓸 수 있다.
  3. 고정 크기 배열의 한계(정해진 칸 수)를 설명하고, 왜 그것이 11장 동적 메모리로 이어지는지 안다.
  4. C의 문자열이 char 배열 + 널 종료('\0') 라는 사실을 이해하고, string.h 함수(strcpy·strlen·strcmp)를 쓸 수 있다.
  5. fgets 로 공백 포함 문자열을 안전하게 입력받고, scanf 뒤에 남는 입력 버퍼 문제를 해결할 수 있다.
  6. 직원 정보를 병렬 배열로 관리하는 방식의 불편함을 몸으로 느끼고, 그것이 구조체(D 모듈)의 필요성으로 이어짐을 안다.

왜 지금, 배열과 문자열인가

B 모듈에서 우리 회사는 "돌기" 시작했다. 매달 정산이 반복되고, 파산 엔딩까지 있다. 그런데 이 회사엔 아직 두 가지가 없다 — 사람과 이름이다.

직원을 한 명 두려면 변수 몇 개(이름·개발력·창의력·월급·근속)가 필요하다. 그런데 직원은 한 명이 아니라 여럿이다. 변수를 dev1, dev2, dev3, dev4 처럼 번호를 붙여 만들 순 없다 — 직원이 늘 때마다 코드를 고쳐야 하고, "모든 직원의 월급을 더하라" 같은 일을 반복문으로 처리할 수도 없다. 같은 종류의 값 여럿을 하나의 이름으로, 번호(인덱스)로 다루는 도구 — 그것이 배열이다.

이름은 또 다른 문제다. 지금까지 우리가 다룬 값은 숫자뿐이었다. 회사 이름 "EQMENT STUDIO", 게임 제목 "던전 앤 라면" 같은 글자의 나열을 담으려면 새로운 그릇이 필요하다. C에는 문자열 전용 자료형이 따로 없다 — 문자열은 글자(char)를 담은 배열이다. 그래서 문자열은 배열을 배운 바로 다음에 온다.

이 모듈에서 만드는 것. 회사에 이름을 붙이고(문자열 입력), 직원을 최대 4명까지 고용해 목록으로 관리하며(배열), 매달 직원 월급 합계를 지출에 반영하는 — GDD 마일스톤 v0.4 "첫 직원" 이다. 5번째 직원을 뽑으려 하면 "사무실이 좁습니다"라며 거절당하는데, 이 답답함이 11장(동적 메모리)에서 풀린다.

7장 · 배열 — 직원 여럿을 한 이름으로

7.1 배열이란 — 같은 자료형 여럿을 한 묶음으로

배열(array) 은 같은 자료형의 값 여러 개를 연속된 칸에 담고, 인덱스(번호) 로 각 칸에 접근하는 자료구조다. 직원 4명의 개발력을 담아 보자.

int staffDev[4];       /* int 4칸짜리 배열 — staffDev[0] ~ staffDev[3] */

staffDev[0] = 7;       /* 첫 번째 직원의 개발력 */
staffDev[1] = 5;
staffDev[2] = 8;
staffDev[3] = 6;

printf("%d\n", staffDev[2]);   /* 8 — 세 번째 직원 */

핵심 규칙 하나 — 인덱스는 0부터 시작한다. 4칸 배열의 유효한 인덱스는 0, 1, 2, 3 이다. staffDev[4] 는 존재하지 않는 다섯 번째 칸으로, 접근하면 프로그램이 엉뚱하게 동작하거나 죽는다(이를 "배열 범위 벗어남"이라 한다). "칸이 4개면 인덱스는 0~3" 을 몸에 새기자.

7.2 초기화와 순회 — for 와 배열은 단짝

배열은 선언과 동시에 값을 채울 수 있고(초기화), for 로 전체를 훑을 수 있다(순회). 배열과 반복문(B 모듈)은 단짝이다.

int sales[4] = { 820, 640, 0, 1200 };   /* 선언과 동시에 초기화 */
int i, total = 0;

for (i = 0; i < 4; i++)      /* 0,1,2,3 — 모든 칸을 순회 */
    total += sales[i];       /* 각 칸의 값을 더한다 */

printf("합계 %d\n", total);   /* 2660 */

for (i = 0; i < 4; i++) — 인덱스 i 를 0부터 3까지 돌리며 sales[i] 로 각 칸에 닿는다. "모든 직원의 월급을 더하라", "모든 매출을 합하라" 같은 일이 전부 이 한 줄짜리 패턴이다. 배열이 없었다면 sales1 + sales2 + sales3 + sales4 처럼 손으로 다 써야 했을 것이다.

4 를 여기저기 쓰지 말자. 배열 크기를 #define MAX_STAFF 4 같은 상수로 두면, 나중에 슬롯을 늘릴 때 한 줄만 고치면 된다. A 모듈의 const RENT 와 같은 정신이다. 우리 게임도 START_STAFF_CAP 이라는 상수로 이 크기를 관리한다.

7.3 2차원 배열 — 표처럼 생긴 배열

배열의 각 칸이 다시 배열이면 2차원 배열이 된다. 행과 열이 있는 표를 생각하면 된다. 우리 게임에선 두 곳에 쓰인다 — 월별 매출 12칸(1차원이면 충분하지만 연도별로 쌓으면 2차원)과, 곧 볼 직원 이름 목록(문자열이 곧 char 배열이라, 이름 여럿은 2차원 char 배열)이다.

int monthSales[3][12];        /* 3년 × 12개월 매출표 */
monthSales[0][2] = 820;       /* 1년차 3월 매출 (행 0, 열 2) */

/* 표 전체 순회 — 이중 for */
int y, m;
for (y = 0; y < 3; y++)
    for (m = 0; m < 12; m++)
        monthSales[y][m] = 0; /* 전부 0으로 초기화 */

2차원 배열의 순회는 for 안에 for 를 넣는 이중 반복문이다. 바깥 for 가 행(년)을, 안쪽 for 가 열(월)을 훑는다. 이 이중 순회 패턴은 2부에서 테트리스 보드(board[20][10])나 미로 격자를 다룰 때 그대로 다시 쓰인다.

7.4 고정 크기의 한계 — 4명이 꽉 차면

배열의 크기는 선언할 때 정해지고, 실행 중에는 바꿀 수 없다. int staffDev[4] 는 영원히 4칸이다. 우리 회사는 처음에 직원을 4명까지만 둘 수 있는데, 이 한계가 곧 게임의 벽이 된다.

#define MAX_STAFF 4
int staffCount = 0;   /* 현재 직원 수 */

/* 고용 시도 */
if (staffCount >= MAX_STAFF) {
    printf("  ! 사무실이 좁습니다. 더는 고용할 수 없어요.\n");
} else {
    /* staffCount 번째 칸에 새 직원을 넣고 staffCount++ */
}

5번째 직원을 뽑으려 하면 거절당한다. 회사는 계속 커져야 하는데 배열은 4칸에 못 박혀 있다 — 이 답답함이 11장 동적 메모리(malloc/realloc) 의 동기다. 그때 우리는 "필요한 만큼 배열을 키우는" 법을 배워 이 벽을 허문다. 지금은 벽에 부딪히는 경험 자체가 학습이다.

⚠️ 고정 배열 → 동적 배열, 커리큘럼의 복선. "정해진 크기의 배열이 부족하다"는 이 불편함을 지금 확실히 느껴 두자. 11장에서 realloc 으로 배열을 2배씩 늘려 이 한계를 푸는데, 불편을 먼저 겪은 사람만이 그 해법의 고마움을 안다. 참고로 2부의 Galaga(오브젝트 풀)는 정반대 방향 — "동적 할당이 너무 잦아 느려지니, 고정 배열을 재활용하자"는 이야기다. 같은 배열이라도 상황에 따라 고정과 동적을 오간다.

7.5 게임에 적용 — 직원 목록을 "병렬 배열"로

이제 직원 목록을 만든다. 직원 한 명은 이름·개발력·창의력·월급·근속 다섯 가지 정보를 가진다. 그런데 아직 구조체가 없으니(9장), 이 다섯 가지를 각각 별도의 배열로 만들어 인덱스로 짝을 맞춘다. 이것을 병렬 배열(parallel array) 이라 한다.

#define MAX_STAFF 4
#define SALARY_RATE 30

char staffName[MAX_STAFF][20];  /* 이름 4명분 (2차원 char = 문자열 4개) */
int  staffDev[MAX_STAFF];       /* 개발력 4명분 */
int  staffCre[MAX_STAFF];       /* 창의력 4명분 */
int  staffSalary[MAX_STAFF];    /* 월급 4명분 */
int  staffMonths[MAX_STAFF];    /* 근속 4명분 */
int  staffCount = 0;            /* 현재 직원 수 */

staffName[2], staffDev[2], staffCre[2] … 인덱스 2가 곧 "세 번째 직원" 이다. 다섯 배열이 인덱스로 손을 잡고 한 직원을 표현한다. 고용은 staffCount 번째 칸을 채우고 수를 늘린다.

/* 고용: 이름·능력치를 받아 다음 빈 칸에 넣는다 */
void HireStaff(const char *name, int dev, int cre)
{
    if (staffCount >= MAX_STAFF) {
        printf("  ! 사무실이 좁습니다. (최대 %d명)\n", MAX_STAFF);
        return;
    }
    strcpy(staffName[staffCount], name);          /* 문자열은 strcpy 로 (8장) */
    staffDev[staffCount]    = dev;
    staffCre[staffCount]    = cre;
    staffSalary[staffCount] = (dev + cre) * SALARY_RATE;  /* 월급 = (개발+창의)*30 */
    staffMonths[staffCount] = 0;
    staffCount++;
    printf("  * [%s] 고용! (월급 %d만원)\n", name, staffSalary[staffCount - 1]);
}

월급 합계는 배열 순회 한 번이다(7.2 패턴 그대로).

int TotalSalary(void)
{
    int i, sum = 0;
    for (i = 0; i < staffCount; i++)   /* staffCount 명만 순회 */
        sum += staffSalary[i];
    return sum;
}

문제는 해고다. 가운데 직원 한 명을 빼면, 뒤 직원들을 한 칸씩 앞으로 당겨야 한다. 그런데 배열이 다섯 개니, 다섯 배열을 전부 당겨야 한다.

/* idx 번 직원 해고: 뒤 직원들을 한 칸씩 앞으로 — 다섯 배열 모두! */
void FireStaff(int idx)
{
    int i;
    for (i = idx; i < staffCount - 1; i++) {
        strcpy(staffName[i], staffName[i + 1]);  /* ① 이름 당기기 */
        staffDev[i]    = staffDev[i + 1];        /* ② 개발력 당기기 */
        staffCre[i]    = staffCre[i + 1];        /* ③ 창의력 당기기 */
        staffSalary[i] = staffSalary[i + 1];     /* ④ 월급 당기기 */
        staffMonths[i] = staffMonths[i + 1];     /* ⑤ 근속 당기기 */
    }
    staffCount--;
}

바로 이것이 병렬 배열의 아픔이다. 직원 하나를 옮기는 데 다섯 줄이 필요하고, 하나라도 빠뜨리면 이름과 능력치가 어긋난다(김개발의 이름 자리에 이서버의 월급이 붙는 식으로). 직원에게 속성 하나(예: 사기)를 추가하면? 배열 선언·고용·해고·순회를 전부 손봐야 한다.

⚠️ 이 불편함을 기억하라 — D 모듈이 이걸 고친다. 다섯 배열을 나란히 당기는 이 코드는 다음 D 모듈(구조체)의 도입부에 그대로 다시 등장한다. 구조체로 직원을 하나로 묶으면, 이 다섯 줄이 staff[i] = staff[i+1]; 한 줄로 접힌다. 지금의 아픔이 클수록, 그때의 후련함도 크다. 이건 우연이 아니라 이 책이 문법을 배우는 순서 — "왜 필요한가를 몸으로 겪은 뒤에 도구를 쥔다" — 그 자체다.

8장 · 문자열 — 회사에 이름을 주다

숫자만 다루던 우리 게임에 글자를 들인다. 회사 이름, 게임 제목, 직원 이름 — 전부 문자열이다.

8.1 문자열은 char 배열 + 널 종료

C에는 문자열 전용 자료형이 없다. 문자열은 글자(char)를 담은 배열이고, 그 끝에는 반드시 널 문자 '\0' 이 붙는다. 이 널 문자가 "문자열이 여기서 끝난다"는 표시다.

char name[6] = "CSGP";

메모리에는 이렇게 담긴다 — 글자 4개 뒤에 자동으로 '\0' 이 붙는다.

  인덱스:  0    1    2    3    4     5
          'C'  'S'  'G'  'P' '\0'  (미사용)
                                ▲
                       여기서 문자열 끝. printf 는 '\0' 을 만나면 멈춘다.

그래서 배열 크기는 글자 수 + 1(널 문자 자리)이어야 한다. "CSGP"(4글자)를 담으려면 최소 char[5] 가 필요하다. 회사명을 char companyName[20] 으로 잡으면, 널 문자를 빼고 최대 19글자까지 담을 수 있다.

왜 '\0' 이 중요한가. printf("%s", name) 은 배열의 시작부터 글자를 찍다가 '\0' 을 만나면 멈춘다. 만약 널 문자가 없으면, printf 는 배열 경계를 넘어 엉뚱한 메모리를 계속 읽으며 쓰레기 값을 쏟아낸다. 문자열을 직접 조작할 때 널 문자를 챙기는 것은 C 문자열의 철칙이다.

8.2 string.h — 문자열 도구 상자

문자열은 배열이라, name = "CSGP"; 같은 대입이 안 된다(배열 이름은 통째로 대입할 수 없다). 대신 string.h 의 함수들을 쓴다.

함수 하는 일 예
strcpy(dst, src) src 를 dst 로 복사 strcpy(name, "CSGP");
strlen(s) 글자 수 ('\0' 제외) strlen("CSGP") → 4
strcmp(a, b) 두 문자열 비교 (같으면 0) strcmp(a, "exit") == 0
strcat(dst, src) dst 뒤에 src 를 이어붙임 strcat(path, ".txt"); 
#include <string.h>

char company[20];
strcpy(company, "EQMENT STUDIO");     /* 대입 대신 복사 */
printf("이름 길이: %d\n", (int)strlen(company));  /* 13 */

if (strcmp(company, "EQMENT STUDIO") == 0)   /* == 이 아니라 strcmp! */
    printf("우리 회사가 맞다\n");

⚠️ 문자열 비교는 == 가 아니라 strcmp. if (name == "CSGP") 는 문자열 내용이 아니라 주소를 비교하므로 거의 항상 거짓이다. 내용이 같은지 물으려면 strcmp(name, "CSGP") == 0 이다. strcmp 는 같으면 0을 돌려주므로 == 0 을 붙이는 걸 잊지 말자(이 0의 의미는 헷갈리기 쉬우니 "다른 정도를 재는데, 다름이 0이면 같다"로 외우자).

8.3 문자열 입력의 함정 — scanf 와 버퍼

문자열을 입력받는 건 숫자보다 까다롭다. scanf("%s", name) 은 공백에서 잘린다 — "던전 앤 라면"을 입력하면 "던전"만 담긴다. 공백을 포함한 한 줄을 통째로 받으려면 fgets 를 쓴다.

char title[30];
fgets(title, sizeof(title), stdin);      /* 한 줄 전체 (공백 포함) */
title[strcspn(title, "\n")] = '\0';      /* 끝의 개행 문자를 널로 바꿔 제거 */

fgets 는 개행(\n)까지 함께 읽어 오므로, strcspn(title, "\n") 으로 개행 위치를 찾아 '\0' 으로 덮어 제거한다. 이 두 줄이 문자열 한 줄을 안전하게 받는 표준 패턴이다.

그런데 여기 C 입문의 가장 악명 높은 함정이 있다. scanf("%d") 로 숫자를 받은 직후에 fgets 를 부르면, fgets 가 빈 줄을 읽고 지나가 버린다.

int sel;
char name[20];
scanf("%d", &sel);              /* 숫자 입력: 사용자가 "3⏎" 입력 */
fgets(name, sizeof(name), stdin); /* ← "3" 뒤에 남은 '\n' 을 읽고 즉시 끝나 버림! */

왜일까? scanf("%d") 는 숫자 3 만 읽고, 사용자가 누른 Enter('\n')를 입력 버퍼에 남겨 둔다. 그 직후 fgets 는 버퍼에 남은 그 '\n' 을 "빈 줄"로 읽어 버린다. 해결책은 scanf 다음에 버퍼에 남은 개행을 비워 주는 것이다.

int ch;
while ((ch = getchar()) != '\n' && ch != EOF)   /* 남은 문자를 개행까지 다 버린다 */
    ;                                            /* 비어 있는 몸통 (버리기만) */

이 while (getchar() != '\n'); 패턴은 "입력 버퍼 비우기"의 상투어다. 실제 게임의 ui.c 는 이걸 FlushLine() 이라는 함수로 묶어, 모든 입력 함수가 공유한다(워크스루에서 본다).

8.4 게임에 적용 — 회사명과 게임 제목

이제 회사에 이름을 붙인다. 타이틀 화면에서 회사명을, 게임 기획에서 제목을 입력받는다. 8.3의 안전한 입력 패턴을 함수로 묶어 재사용한다.

#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>
#include <string.h>

char companyName[20];
char gameTitle[30];

/* 한 줄 입력 (공백 포함, 개행 제거) — 8.3 패턴을 함수로 */
void ReadLine(const char *prompt, char *buf, int size)
{
    printf("%s", prompt);
    fgets(buf, size, stdin);
    buf[strcspn(buf, "\n")] = '\0';   /* 개행 제거 → 깔끔한 문자열 */
}

int main(void)
{
    ReadLine("회사 이름: ", companyName, sizeof(companyName));
    ReadLine("첫 게임 제목: ", gameTitle, sizeof(gameTitle));

    printf("\n[%s] 창업! 첫 작품 \"%s\" 개발을 시작합니다.\n",
           companyName, gameTitle);
    printf("회사명 %d글자 / 제목 %d글자\n",
           (int)strlen(companyName), (int)strlen(gameTitle));
    return 0;
}

실행하면 공백이 든 이름도 온전히 담긴다.

회사 이름: EQMENT STUDIO
첫 게임 제목: 던전 앤 라면
[EQMENT STUDIO] 창업! 첫 작품 "던전 앤 라면" 개발을 시작합니다.
회사명 13글자 / 제목 ... (한글은 글자당 여러 바이트라 strlen 값이 눈에 보이는 글자 수와 다르다)

한글과 strlen — 인코딩의 그림자. strlen 은 바이트 수를 센다. CP949에서 한글 한 글자는 2바이트라, "던전 앤 라면"의 strlen 은 눈에 보이는 글자 수보다 크다. 지금은 "한글은 한 글자가 여러 바이트"라는 사실만 알아 두자. 이 바이트-글자 불일치가 2부에서 전각 문자가 콘솔 2칸을 차지하는 문제(x * 2 좌표 변환)로 다시 나타난다. 인코딩은 1부와 2부를 관통하는 그림자 주제다.

코드 워크스루 — 실제 소스에서 보는 배열과 문자열

우리가 병렬 배열로 만든 직원 목록이, Study/Tycoon/ 완성본에서는 어떻게 나타나는지 보자. 완성본은 이미 구조체(D 모듈)로 넘어가 있어서, 병렬 배열이 하나의 구조체 배열로 접혀 있다 — 우리가 다음 모듈에서 할 바로 그 개선이 완성본엔 이미 되어 있다.

실제 파일: Study/Tycoon/employee.c(직원 목록) · Study/Tycoon/ui.c(문자열 입력)

① 월급 합계 (employee.c · TotalSalary) — 우리 7.5의 TotalSalary 와 판박이다. 배열을 for 로 순회하며 더한다. 다른 점은 다섯 배열이 아니라 하나의 배열(c->staff[i])에서 .salary 를 꺼낸다는 것뿐.

int TotalSalary(const Company *c)
{
    int sum = 0, i;
    for (i = 0; i < c->staffCount; i++)
        sum += c->staff[i].salary;      /* 우리의 staffSalary[i] 에 해당 */
    return sum;
}

우리는 staffSalary[i](다섯 배열 중 하나), 완성본은 c->staff[i].salary(한 배열 안의 멤버). 순회 패턴은 똑같고, 데이터를 묶는 방식만 다르다. 그 "묶는 방식"의 차이가 바로 D 모듈이다.

② 해고 (employee.c · QuitEmployee) — 우리 7.5의 FireStaff 는 다섯 배열을 각각 당기느라 다섯 줄이었다. 완성본은 구조체 덕분에 단 한 줄이다. 이 대비를 꼭 눈에 새기자.

void QuitEmployee(Company *c, int idx)
{
    int i;
    for (i = idx; i < c->staffCount - 1; i++)
        c->staff[i] = c->staff[i + 1];   /* 구조체 통째 복사 — 우리의 5줄이 1줄로! */
    c->staffCount--;
}

우리의 다섯 줄(이름·개발·창의·월급·근속을 따로따로 당기기)이, 여기선 c->staff[i] = c->staff[i + 1]; 한 줄로 끝난다. 구조체가 다섯 값을 하나로 묶었기 때문이다. 7.5에서 느낀 그 아픔의 해답이 이 한 줄에 있다.

③ 문자열 입력 (ui.c · ReadLine) — 우리 8.4의 ReadLine 을 더 튼튼하게 만든 실전 버전이다. 8.3에서 배운 개행 제거와 버퍼 처리가 모두 들어 있다.

void ReadLine(const char *prompt, char *buf, int size)
{
    /* ... fgets 로 한 줄 읽기 ... */
    if (len > 0 && buf[len - 1] != '\n')
        FlushLine();                     /* 줄이 너무 길면 버퍼에 남은 걸 비운다 */
    buf[strcspn(buf, "\n")] = '\0';      /* 개행 제거 (우리 8.3 그대로) */
    if (buf[0] != '\0')
        return;                          /* 빈 줄이면 다시 입력받는다 */
}

FlushLine() 이 8.3에서 배운 while (getchar() != '\n'); 를 함수로 묶은 것이다. 완성본은 여기에 "입력이 너무 길 때"와 "빈 줄일 때"까지 방어를 더했다 — 실전 입력 처리는 이렇게 예외를 하나씩 챙기는 일이다.

그림으로 정리 — 병렬 배열과 문자열

📊 웹 교육문서에서는 병렬 배열의 해고 과정이 애니메이션(V-P1-C)으로 재생된다. 다섯 배열이 나란히 한 칸씩 밀리는 모습을, 하나만 어긋나면 데이터가 뒤섞이는 위험과 함께 보여 준다. 책에서는 아래 도식으로 대신한다.

병렬 배열 — 한 직원이 다섯 배열에 흩어져 있다

  인덱스:      [0]      [1]      [2]      [3]
  staffName:  김개발   이서버   박퍼즈   (빈칸)
  staffDev:     7        5        8       0
  staffCre:     5        6        4       0
  staffSalary: 360      330      360      0
  staffMonths:  3        1        0       0
               └── 한 직원 ──┘
   ⚠ [1] 이서버를 해고하면 [2][3]을 다섯 배열 모두 한 칸씩 당겨야 한다.
      하나라도 빠뜨리면 이름과 능력치가 어긋난다.
   → D 모듈: 이 다섯 배열을 struct Employee staff[4] 하나로 묶는다.

문자열 — char 배열 끝의 '\0'

  char title[8] = "CSGP";
  ┌────┬────┬────┬────┬────┬────┬────┬────┐
  │ 'C'│ 'S'│ 'G'│ 'P'│'\0'│ ?  │ ?  │ ?  │
  └────┴────┴────┴────┴────┴────┴────┴────┘
    0    1    2    3    4    5    6    7
                        ▲
             널 문자 = "여기서 끝". strlen 은 4, 배열은 8칸.
             printf("%s") 는 '\0' 을 만날 때까지만 찍는다.

한 문장으로: 배열은 같은 것을 번호로 줄 세운 것이고, 문자열은 글자를 줄 세운 뒤 끝에 '\0' 이라는 마침표를 찍은 것이다.

체크포인트 — 스스로 점검하기

Q1. int a[4]; 에서 유효한 인덱스는? a[4] 에 값을 넣으면 왜 위험한가?

Q2. char s[4] = "CSGP"; 는 왜 문제가 되는가? (힌트: 널 문자)

Q3. 두 문자열이 같은지 검사하려는데 if (name == "kim") 이 항상 거짓이다. 왜이고, 어떻게 고치나?

Q4. scanf("%d", &n); 바로 뒤에 fgets(buf, ...) 를 부르면 buf 가 빈 줄이 된다. 왜인가?

Q5. 직원을 병렬 배열 5개로 관리할 때, 가운데 직원 한 명을 해고하는 코드가 하기 번거로운 이유는? 이 문제를 어떻게 풀 수 있을까?

해설 펼치기

A1. 유효한 인덱스는 0, 1, 2, 3(0부터 크기−1까지). a[4] 는 배열의 다섯 번째 칸으로 존재하지 않는 메모리다. 거기에 쓰면 다른 변수를 덮어쓰거나 프로그램이 죽는 "배열 범위 벗어남" 버그가 된다.

A2. "CSGP"는 4글자지만 끝에 널 문자 '\0' 이 붙어 5바이트가 필요하다. char s[4] 는 4칸뿐이라 널 문자를 담을 자리가 없다. char s[5](또는 크기를 비워 char s[] = "CSGP";)로 해야 안전하다.

A3. == 는 문자열 내용이 아니라 주소(포인터 값)를 비교한다. 내용이 같아도 주소가 다르면 거짓이다. strcmp(name, "kim") == 0 으로 내용을 비교해야 한다(strcmp 는 같으면 0을 반환).

A4. scanf("%d") 는 숫자만 읽고, 사용자가 누른 Enter('\n')를 입력 버퍼에 남겨 둔다. 그 직후 fgets 가 버퍼에 남은 그 개행을 읽어 "빈 줄"로 처리해 버린다. scanf 다음에 while (getchar() != '\n'); 로 버퍼를 비워야 한다.

A5. 직원 하나의 정보가 다섯 배열에 흩어져 있어서, 한 명을 당기려면 다섯 배열을 모두 한 칸씩 당겨야 하고, 하나라도 빠뜨리면 이름과 능력치가 어긋난다. 속성을 추가하면 손볼 곳이 또 늘어난다. 해법은 다섯 값을 하나로 묶는 것 — 그것이 다음 D 모듈의 구조체다. 구조체를 쓰면 staff[i] = staff[i+1]; 한 줄로 해결된다.

실습 미션 — 3목(틱택토) tictactoe.c ⭐

본편(타이쿤)이 "한 게임을 계속 키우는 세로 축"이라면, 실습 미션은 이 모듈의 새 문법만 콕 집어 연습하는 독립 미니게임(가로 축) 이다. 먼저 직접 짜 보고, 막히면 힌트를, 그래도 막히면 정답을 펼쳐 대조하자.

드릴 문법 — 2차원 배열 char board[3][3] · 배열 초기화·순회(이중 for) · 문자열 배열(플레이어 이름) · 인덱스 접근 만드는 것 — 두 사람이 번갈아 O/X를 놓는 3×3 틱택토. 한 줄을 먼저 완성하면 승리. 클리어 조건 — 매 턴 좌표를 입력받아 말을 놓고, 승리/무승부를 판정해 출력한다. 규모 — 약 60줄, 파일 1개. 표준 C만. B 모듈의 반복문과 이 모듈의 2차원 배열이 만나는 미션이다.

문제 정의

이 게임의 메인 자료구조는 2차원 배열 char board[3][3] 하나다. 빈칸은 ' ', 놓인 말은 'O'/'X' 로 담는다. 보드를 그리는 것도(이중 for 순회), 말을 놓는 것도(인덱스 접근), 승리를 확인하는 것도(줄마다 세 칸 비교) 전부 이 배열을 다루는 일이다. 2부의 테트리스 보드(board[20][10])를 미리 맛보는 미션이기도 하다.

목표 / 클리어 조건

  1. char board[3][3] 를 ' ' 로 초기화한다(이중 for).
  2. 두 플레이어 이름을 문자열 배열 char names[2][20] 에 입력받는다.
  3. 번갈아 좌표(행·열)를 입력받아 말을 놓는다. 이미 찬 칸이면 다시.
  4. 매 턴 후 승리(가로·세로·대각선 세 칸이 같은지)를 판정한다.
  5. 승자가 나오거나 9칸이 다 차면(무승부) 게임 종료.

힌트

힌트 1 — 보드 초기화와 출력 
char board[3][3];
int r, c;
for (r = 0; r < 3; r++)
    for (c = 0; c < 3; c++)
        board[r][c] = ' ';        /* 이중 for 로 전부 빈칸 */

/* 출력 */
for (r = 0; r < 3; r++) {
    for (c = 0; c < 3; c++)
        printf(" %c ", board[r][c]);
    printf("\n");
}
힌트 2 — 승리 판정 함수

세 칸이 같고 빈칸이 아니면 승리. 8줄(가로3·세로3·대각2)을 검사한다.

int CheckWin(char b[3][3], char mark)
{
    int i;
    for (i = 0; i < 3; i++) {
        if (b[i][0]==mark && b[i][1]==mark && b[i][2]==mark) return 1; /* 가로 */
        if (b[0][i]==mark && b[1][i]==mark && b[2][i]==mark) return 1; /* 세로 */
    }
    if (b[0][0]==mark && b[1][1]==mark && b[2][2]==mark) return 1;     /* 대각 ↘ */
    if (b[0][2]==mark && b[1][1]==mark && b[2][0]==mark) return 1;     /* 대각 ↙ */
    return 0;
}

(배열을 함수에 넘길 때 char b[3][3] 로 받는다 — 배열은 사실상 주소로 전달되므로 함수 안에서 원본이 바뀐다. 이 "배열=주소" 감각이 D 모듈 포인터의 예고편이다.)

힌트 3 — 턴 루프

for (turn = 0; turn < 9; turn++) 로 최대 9턴. turn % 2 로 현재 플레이어(0/1)를 정하고, 놓은 뒤 CheckWin 으로 승리를 확인해 승리면 break.

정답 코드

정답 펼치기 — tictactoe.c

void PrintBoard(char b[3][3])
{
    int r, c;
    printf("\n");
    for (r = 0; r < 3; r++) {
        for (c = 0; c < 3; c++) {
            printf(" %c ", b[r][c]);
            if (c < 2) printf("|");
        }
        printf(r < 2 ? "\n---+---+---\n" : "\n");
    }
}

int CheckWin(char b[3][3], char mark)
{
    int i;
    for (i = 0; i < 3; i++) {
        if (b[i][0]==mark && b[i][1]==mark && b[i][2]==mark) return 1;
        if (b[0][i]==mark && b[1][i]==mark && b[2][i]==mark) return 1;
    }
    if (b[0][0]==mark && b[1][1]==mark && b[2][2]==mark) return 1;
    if (b[0][2]==mark && b[1][1]==mark && b[2][0]==mark) return 1;
    return 0;
}

int main(void)
{
    char board[3][3];
    char names[2][20];              /* 문자열 배열 — 두 플레이어 이름 */
    char marks[2] = { 'O', 'X' };
    int r, c, turn, p, won = 0;

    for (r = 0; r < 3; r++)
        for (c = 0; c < 3; c++)
            board[r][c] = ' ';      /* 2차원 배열 초기화 */

    printf("1P(O) 이름: ");  fgets(names[0], 20, stdin);
    printf("2P(X) 이름: ");  fgets(names[1], 20, stdin);

    for (turn = 0; turn < 9; turn++) {
        p = turn % 2;               /* 0 = 1P, 1 = 2P */
        PrintBoard(board);
        printf("[%c] 차례. 행 열 (0~2): ", marks[p]);

        if (scanf("%d %d", &r, &c) != 2) { FlushLine(); continue; }
        FlushLine();

        if (r < 0 || r > 2 || c < 0 || c > 2 || board[r][c] != ' ') {
            printf("  ! 놓을 수 없는 칸입니다.\n");
            turn--;                 /* 이번 턴 무효 → 같은 사람 다시 */
            continue;
        }
        board[r][c] = marks[p];     /* 말을 놓는다 */

        if (CheckWin(board, marks[p])) {
            PrintBoard(board);
            printf(">> [%c] 승리!\n", marks[p]);
            won = 1;
            break;
        }
    }

    if (!won) { PrintBoard(board); printf(">> 무승부!\n"); }
    return 0;
}

빌드: cl tictactoe.c (MSVC) 또는 gcc tictactoe.c -o tictactoe

함정/체크포인트

  • 2차원 배열 초기화를 빠뜨리면 board 에 쓰레기 값이 들어 있어, 빈칸 검사(board[r][c] != ' ')가 엉뚱하게 동작한다. 이중 for 로 반드시 ' ' 로 채우고 시작한다.
  • char b[3][3] 로 배열을 함수에 넘겨도 함수 안에서 놓은 말이 main 의 원본에 반영된다 — 배열은 값이 아니라 주소로 전달되기 때문이다. 이 사실이 D 모듈 "포인터"의 핵심 감각으로 이어진다.
  • scanf 와 이름 입력을 섞으면 8.3의 버퍼 문제가 나온다. FlushLine() 으로 개행을 비워 주는 이유를 코드에서 확인하자.

확장 과제

  • E 모듈(난수) 에서 2P를 컴퓨터로 바꿔, 빈칸 중 하나를 rand 로 골라 두게 한다.
  • 보드를 board[N][N] 과 승리 조건 K 로 일반화해 "N목"으로 확장(2부 미로의 격자 좌표계 감각).

이 모듈을 마치며

얻은 것 다음에 쓰는 곳
배열로 같은 것 여럿 담기 직원 목록, 출시작 목록, 2부 테트리스 보드
배열 순회(for)로 합계·검색 월급 합계, 능력치 합산(D 모듈), 판매 정산
문자열(char 배열 + '\0') 회사명·게임 제목·직원 이름, 세이브 파일(F 모듈)
병렬 배열의 불편함 D 모듈 구조체 — 다섯 배열을 하나로 묶는 동기

우리는 회사에 사람(직원 배열)과 이름(문자열)을 채웠다. 하지만 직원 하나를 표현하려 배열 다섯 개를 나란히 끌고 다니는 지금 방식은, 손이 아프고 실수가 잦다. 다음 D 모듈에서 이 다섯 배열을 구조체 하나로 묶고, 그것을 함수에 포인터로 넘겨 직접 고치게 만든다. 1부의 가장 큰 고비이자, 게임의 뼈대가 완성되는 지점이다.
